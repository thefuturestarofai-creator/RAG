# 文件操作

> RAG 系统的第一步就是**读取文档**。无论是 txt、csv、json 还是 pdf，最终都要被 Python 读进来变成字符串或字典，才能进行后续的切分和向量化。

---

## 1. 用"读信"来理解文件操作

文件操作就像**读信和写信**：

- **打开信封**（`open`）→ **读/写内容**（`read`/`write`）→ **封好信封**（`close`）
- 忘记封信封（忘记 `close`）→ 可能丢数据
- 用 `with` 语句 → 自动帮你封信封，不用担心忘记

---

## 2. 最基本的文件读写

### 2.1 读取文件

```python
# 最基本的方式（容易忘记 close，不推荐）
file = open("data.txt", "r", encoding="utf-8")
content = file.read()
file.close()

# 推荐方式：with 语句（自动关闭文件）
with open("data.txt", "r", encoding="utf-8") as file:
    content = file.read()
# 离开 with 块后，文件自动关闭
```

### 2.2 写入文件

```python
# "w" 模式 = 覆盖写入（文件不存在则创建）
with open("output.txt", "w", encoding="utf-8") as file:
    file.write("第一行内容\n")
    file.write("第二行内容\n")

# "a" 模式 = 追加写入（不覆盖原有内容）
with open("output.txt", "a", encoding="utf-8") as file:
    file.write("追加的一行\n")
```

### 2.3 文件打开模式

| 模式 | 含义 | 使用场景 |
|------|------|----------|
| `"r"` | 只读（默认） | 读取已有文件 |
| `"w"` | 覆盖写入 | 创建新文件或覆盖旧文件 |
| `"a"` | 追加写入 | 在文件末尾添加内容 |
| `"r+"` | 读写 | 同时读取和写入 |

## 3. 按行读取（RAG 中最常用）

处理大文件时，不要一次性 `read()` 全部内容，应该按行读取：

```python
# 方法1：readlines() —— 读取所有行到列表
with open("data.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()    # 返回 ["第一行\n", "第二行\n", ...]
    for line in lines:
        print(line.strip())  # strip() 去掉末尾的换行符

# 方法2：直接遍历文件对象（推荐，内存友好）
with open("data.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())

# 方法3：readline() —— 每次读一行
with open("data.txt", "r", encoding="utf-8") as f:
    line = f.readline()      # 读一行
    while line:
        print(line.strip())
        line = f.readline()  # 读下一行
```

## 4. 读取常见文件格式

### 4.1 读取 TXT 文件

```python
with open("document.txt", "r", encoding="utf-8") as f:
    text = f.read()
print(f"文档长度: {len(text)} 字符")
```

### 4.2 读取 CSV 文件

```python
import csv

# 读取 CSV
with open("data.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)    # 每行变成一个字典
    for row in reader:
        print(row["name"], row["content"])

# 写入 CSV
with open("output.csv", "w", encoding="utf-8", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["name", "content"])
    writer.writeheader()
    writer.writerow({"name": "文档1", "content": "这是内容"})
```

### 4.3 读取 JSON 文件

```python
import json

# 读取 JSON
with open("config.json", "r", encoding="utf-8") as f:
    config = json.load(f)         # 直接变成 Python 字典
print(config["api_key"])

# 写入 JSON
data = {"chunks": ["第一段", "第二段"], "count": 2}
with open("output.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
# ensure_ascii=False 让中文正常显示
# indent=2 让输出格式化，便于阅读
```

## 5. 路径处理：`os.path` 和 `pathlib`

### 5.1 用 `os.path` 处理路径

```python
import os

# 拼接路径（跨平台兼容）
path = os.path.join("data", "documents", "file.txt")

# 判断文件/目录是否存在
os.path.exists("data.txt")          # True/False
os.path.isfile("data.txt")          # 是否是文件
os.path.isdir("data")               # 是否是目录

# 获取文件信息
os.path.getsize("data.txt")         # 文件大小（字节）
os.path.basename("/a/b/file.txt")   # "file.txt"（文件名）
os.path.dirname("/a/b/file.txt")    # "/a/b"（目录名）
os.path.splitext("file.txt")        # ("file", ".txt")（分离扩展名）

# 创建目录
os.makedirs("output/results", exist_ok=True)  # exist_ok=True 目录已存在不报错
```

### 5.2 用 `pathlib` 处路径（更现代的方式）

```python
from pathlib import Path

# 创建路径对象
p = Path("data") / "documents" / "file.txt"

# 常用操作
p.exists()             # 是否存在
p.is_file()            # 是否是文件
p.read_text()          # 读取文本内容
p.write_text("内容")   # 写入文本
p.suffix               # ".txt"（扩展名）
p.stem                 # "file"（不含扩展名的文件名）
p.parent               # 父目录

# 遍历目录下所有文件
for file in Path("data").glob("*.txt"):
    print(file)

# 递归遍历
for file in Path("data").rglob("*.md"):
    print(file)
```

## 6. 文件操作中的编码问题

中文文件最常见的问题就是**编码错误**：

```python
# 常见错误：UnicodeDecodeError
# 原因：文件不是 UTF-8 编码

# 解决方案1：指定正确的编码
with open("file.txt", "r", encoding="gbk") as f:    # GBK 编码
    content = f.read()

# 解决方案2：忽略无法解码的字符
with open("file.txt", "r", encoding="utf-8", errors="ignore") as f:
    content = f.read()

# 解决方案3：自动检测编码（需要 chardet 库）
# pip install chardet
import chardet
with open("file.txt", "rb") as f:
    result = chardet.detect(f.read())
    encoding = result["encoding"]
with open("file.txt", "r", encoding=encoding) as f:
    content = f.read()
```

## 7. 在 RAG 项目中的应用场景

| 文件操作 | RAG 中的具体用途 |
|---------|-----------------|
| 读取 TXT | 加载知识库文档 |
| 读取 CSV | 加载结构化数据（问答对、数据集） |
| 读取 JSON | 加载配置文件、API 响应、向量数据库导出数据 |

| 按行读取 | 逐行处理大文件，避免内存溢出 |

| 编码处理 | 处理中文文档、混合编码的老文件 |

### RAG 文档加载流程

```
原始文件 (.txt/.csv/.json/.pdf)
    ↓  文件操作：读取
原始文本 (str)
    ↓  文本切分
文档块列表 ([str, str, ...])
    ↓  Embedding
向量列表 ([float, float, ...])
    ↓  存入向量数据库
可检索的知识库
```

## 8. 面试官常问问题

**Q1: `with open()` 和直接 `open()` 有什么区别？为什么推荐用 `with`？**

A: `with` 语句会在块结束时**自动关闭文件**，即使中间发生异常也能保证文件被关闭。直接 `open()` 如果忘记 `close()`，可能导致数据丢失或文件句柄泄漏。

**Q2: `read()`、`readline()`、`readlines()` 有什么区别？**

A: `read()` 一次读全部内容返回字符串；`readline()` 一次读一行返回字符串；`readlines()` 读所有行返回列表。大文件用 `readline()` 或直接遍历文件对象，避免内存爆炸。

**Q3: 处理中文文件时遇到 `UnicodeDecodeError` 怎么办？**

A: 指定正确的编码（`encoding="gbk"` 或 `encoding="utf-8"`），或者用 `errors="ignore"` 跳过无法解码的字符。

**Q4: RAG 系统中，如何批量加载一个目录下的所有文档？**

A: 用 `os.listdir()` 或 `pathlib.Path.glob()` 遍历目录，按扩展名过滤，循环读取每个文件。实际项目中 LangChain 的 `DirectoryLoader` 就是这样做的。

**Q5: JSON 和 CSV 分别适合什么场景？**

A: JSON 适合嵌套数据（配置、API 响应），CSV 适合表格数据（扁平结构）。RAG 中配置用 JSON，数据集常用 CSV 或 JSONL。
