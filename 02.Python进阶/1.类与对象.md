# 类与对象

> 类（class）是 Python 面向对象编程的核心。在 LangChain 框架中，Chain、Retriever、LLM、PromptTemplate 全部都是类。理解类和对象，是读懂任何 AI 框架源码的前提。

---

## 1. 什么是类和对象？

### 用"快递柜"来理解

想象一个**快递柜**：

- **类（class）** = 快递柜的设计图纸。图纸规定了：有多少个格子、每个格子多大、怎么开锁、怎么关锁。
- **对象（object）** = 按照图纸造出来的**实体快递柜**。你可以造很多个快递柜（多个对象），每个柜子里放着不同的快递（不同的数据）。

```python
# 类 = 设计图纸
class 快递柜:
    pass

# 对象 = 实体柜子
柜子A = 快递柜()
柜子B = 快递柜()
# 柜子A 和 柜子B 是两个不同的柜子，互不影响
```

**一句话总结：类是模板，对象是根据模板造出来的实例。**

## 2. 定义一个类

```python
class Document:
    """文档类 —— RAG 中每个文档块的基本结构"""

    def __init__(self, content, source):
        """
        __init__ 是"构造方法"，创建对象时自动调用。
        self 代表"当前对象自己"。
        content 和 source 是创建时传入的参数。
        """
        self.content = content    # 把参数存到对象身上
        self.source = source
        self.score = 0.0          # 默认值

    def summary(self):
        """返回文档摘要"""
        return f"[{self.source}] {self.content[:20]}..."
```

**关键概念：**

| 概念 | 解释 | 类比 |
|------|------|------|
| `class` | 定义类的关键字 | 设计图纸的标题 |
| `__init__` | 构造方法，创建对象时自动执行 | 组装快递柜的步骤 |
| `self` | 代表当前对象自身 | "我自己的" |
| `self.xxx` | 对象的属性（数据） | 柜子里放的东西 |
| `def xxx(self)` | 对象的方法（功能） | 柜子能做的事 |

## 3. 创建和使用对象

```python
# 创建对象（实例化）
doc1 = Document("RAG是检索增强生成技术", "intro.pdf")
doc2 = Document("向量数据库用于存储Embedding", "vector.pdf")

# 访问属性
print(doc1.content)       # "RAG是检索增强生成技术"
print(doc1.source)        # "intro.pdf"

# 调用方法
print(doc1.summary())     # "[intro.pdf] RAG是检索增强生成技术..."

# 修改属性
doc1.score = 0.95
print(doc1.score)         # 0.95
```

**注意：`doc1` 和 `doc2` 是两个独立的对象，修改 `doc1.score` 不会影响 `doc2`。**

## 4. 继承：站在父类的肩膀上

继承就像**子承父业**：子类自动拥有父类的所有属性和方法，还可以添加自己的新功能。

### 用"交通工具"来理解

```
交通工具（父类）
├── 有轮子
├── 能移动
│
├── 汽车（子类）—— 继承了轮子和移动，新增"加油"
├── 自行车（子类）—— 继承了轮子和移动，新增"脚踏"
```

### 代码示例

```python
# 父类
class BaseRetriever:
    """检索器基类 —— 所有检索器的共同功能"""

    def __init__(self, top_k=5):
        self.top_k = top_k

    def retrieve(self, query):
        """检索方法（通用逻辑）"""
        print(f"正在检索：{query}，返回 Top {self.top_k} 结果")

# 子类继承父类
class VectorRetriever(BaseRetriever):
    """向量检索器 —— 继承 BaseRetriever，新增向量搜索功能"""

    def __init__(self, top_k=5, model="text-embedding-ada-002"):
        super().__init__(top_k)       # 调用父类的 __init__
        self.model = model            # 子类自己的新属性

    def embed(self, text):
        """子类新增的方法"""
        return f"[{self.model}] {text} → [0.1, 0.2, ...]"

# 使用
retriever = VectorRetriever(top_k=3, model="text-embedding-3-small")
retriever.retrieve("什么是RAG？")        # 继承自父类
print(retriever.embed("Hello"))          # 子类自己的方法
print(retriever.top_k)                   # 继承的属性
```

## 5. 方法：对象能做的事

### 5.1 实例方法（最常用）

```python
class TextSplitter:
    def __init__(self, chunk_size=500):
        self.chunk_size = chunk_size

    def split(self, text):
        """实例方法 —— 操作对象的数据"""
        chunks = []
        for i in range(0, len(text), self.chunk_size):
            chunks.append(text[i:i + self.chunk_size])
        return chunks

splitter = TextSplitter(chunk_size=100)
result = splitter.split("这是一段很长的文本..." * 50)
```

### 5.2 类方法（用 @classmethod 装饰）

```python
class Document:
    def __init__(self, content, doc_type):
        self.content = content
        self.doc_type = doc_type

    @classmethod
    def from_text(cls, text):
        """类方法 —— 用另一种方式创建对象"""
        return cls(text, "text")       # cls 就是 Document 类本身

    @classmethod
    def from_pdf(cls, text, page):
        doc = cls(text, "pdf")
        doc.page = page
        return doc

# 两种创建方式
doc1 = Document("内容", "text")              # 普通方式
doc2 = Document.from_text("内容")             # 类方法方式
doc3 = Document.from_pdf("PDF内容", page=3)   # 类方法方式
```

### 5.3 静态方法（用 @staticmethod 装饰）

```python
class Utils:
    @staticmethod
    def clean_text(text):
        """静态方法 —— 不需要访问对象数据，纯粹的工具函数"""
        return text.strip().replace("\n", " ")

# 不需要创建对象，直接调用
clean = Utils.clean_text("  hello\nworld  ")
```

## 6. 魔术方法（特殊方法）

以双下划线开头和结尾的方法，Python 会在特定场景自动调用。

```python
class Chunk:
    def __init__(self, content, score):
        self.content = content
        self.score = score

    def __repr__(self):
        """打印对象时显示的内容"""
        return f"Chunk(score={self.score}, content='{self.content[:20]}...')"

    def __lt__(self, other):
        """定义"小于"比较，用于排序"""
        return self.score < other.score

    def __len__(self):
        """定义 len() 的返回值"""
        return len(self.content)

# 使用
c1 = Chunk("RAG技术介绍", 0.95)
c2 = Chunk("向量数据库", 0.87)

print(c1)              # Chunk(score=0.95, content='RAG技术介绍...')
chunks = [c1, c2]
chunks.sort()          # 按 score 排序（因为定义了 __lt__）
print(len(c1))         # 调用 __len__
```

## 7. 在 RAG 项目中的应用场景

| 类与对象概念 | RAG 中的具体应用 |
|-------------|-----------------|
| 类的定义 | LangChain 中的 `LLM`、`Retriever`、`Chain`、`PromptTemplate` 都是类 |
| `__init__` | 初始化检索器时设置参数：`top_k`、`model_name`、`chunk_size` 等 |
| 继承 | 自定义 Retriever 继承 `BaseRetriever`，只重写检索逻辑 |
| 方法 | `.retrieve(query)` 检索、`.invoke(input)` 调用、`.load()` 加载数据 |
| 类方法 | `Document.from_file("doc.pdf")` 从文件创建文档对象 |
| 对象组合 | `Chain` 对象内部组合了 `Retriever` + `LLM` + `PromptTemplate` |

### LangChain 源码中的类（简化示意）

```python
# LangChain 就是这样组织代码的（简化版）
class BaseLanguageModel:
    def invoke(self, prompt):
        raise NotImplementedError

class OpenAI(BaseLanguageModel):
    def __init__(self, model="gpt-3.5-turbo", temperature=0.7):
        self.model = model
        self.temperature = temperature

    def invoke(self, prompt):
        return f"[{self.model}] 回答: ..."

class BaseRetriever:
    def retrieve(self, query):
        raise NotImplementedError

class VectorRetriever(BaseRetriever):
    def __init__(self, vectorstore, top_k=5):
        self.vectorstore = vectorstore
        self.top_k = top_k

    def retrieve(self, query):
        return self.vectorstore.similarity_search(query, k=self.top_k)
```

## 8. 面试官常问问题

**Q1: 类和函数有什么区别？什么时候用类，什么时候用函数？**

A: 函数是"做一个动作"，类是"描述一个事物及其行为"。当你需要**保存状态**（记住数据）时用类，单纯的**数据转换**用函数。RAG 中，Retriever 有配置参数需要保存，所以用类；文本清洗只做转换，用函数即可。

**Q2: `self` 是什么？为什么每个方法都要写？**

A: `self` 代表"当前对象自己"。因为 Python 的方法定义在类里，但要作用在具体的对象上，所以需要 `self` 来区分"哪个对象的数据"。调用时 Python 自动传入，不需要手动写：`doc.summary()` 等价于 `Document.summary(doc)`。

**Q3: `__init__` 和普通方法有什么区别？**

A: `__init__` 是构造方法，在 `ClassName()` 创建对象时**自动调用**，用于初始化属性。普通方法需要手动调用。

**Q4: 继承有什么好处？什么时候用继承？**

A: 继承实现了**代码复用**和**多态**。当多个类有共同的功能时，把共同部分提取到父类。LangChain 的 Retriever 就是这样：`BaseRetriever` 定义接口，`VectorRetriever`、`BM25Retriever` 各自实现具体逻辑。

**Q5: 什么是"面向对象"？什么是"面向过程"？**

A: 面向过程 = 按步骤做事（先做A，再做B）；面向对象 = 把数据和操作打包成对象，让对象之间协作。大型项目（如 LangChain）用面向对象更易维护，小脚本用面向过程更简单。
