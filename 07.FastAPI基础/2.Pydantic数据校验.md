# Pydantic 数据校验

## 什么是 Pydantic？

Pydantic 是一个数据验证库，它使用 Python 的类型提示来验证和解析数据。

类比：Pydantic 就像一个**严格的门卫**，它会检查每个进入系统的数据是否符合要求——类型对不对、必填字段有没有、值在不在合理范围内。不符合要求的数据会被直接拒绝，并告诉你哪里有问题。

## 核心概念

### 1. BaseModel

所有数据模型都继承自 `BaseModel`：

```python
from pydantic import BaseModel

class User(BaseModel):
    name: str              # 必填，字符串类型
    age: int               # 必填，整数类型
    email: str             # 必填，字符串类型
    is_active: bool = True # 可选，默认为 True
```

### 2. 字段类型

Pydantic 支持丰富的字段类型：

```python
from pydantic import BaseModel, Field
from typing import Optional, List

class Product(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)  # 字符串，1-100字符
    price: float = Field(..., gt=0)                        # 浮点数，大于0
    quantity: int = Field(default=0, ge=0)                 # 整数，大于等于0
    tags: List[str] = []                                   # 字符串列表
    description: Optional[str] = None                      # 可选字段
```

常用验证参数：
- `...`：必填字段
- `gt`/`ge`：大于/大于等于
- `lt`/`le`：小于/小于等于
- `min_length`/`max_length`：字符串长度限制
- `pattern`：正则表达式匹配

### 3. 请求模型和响应模型

```python
from pydantic import BaseModel

# 请求模型：定义客户端发送的数据格式
class ChatRequest(BaseModel):
    question: str
    context: str = ""

# 响应模型：定义服务端返回的数据格式
class ChatResponse(BaseModel):
    answer: str
    sources: list[str] = []

# 在路由中使用
@app.post("/chat", response_model=ChatResponse)
async def chat(request: ChatRequest):
    return ChatResponse(
        answer="这是回答",
        sources=["文档1", "文档2"]
    )
```

### 4. 嵌套模型

```python
from pydantic import BaseModel
from typing import List

class Tag(BaseModel):
    name: str
    color: str = "blue"

class Article(BaseModel):
    title: str
    content: str
    tags: List[Tag] = []  # 嵌套模型

# 使用
article = Article(
    title="RAG教程",
    content="...",
    tags=[{"name": "AI"}, {"name": "Python", "color": "red"}]
)
```

### 5. 模型方法

```python
user = User(name="Alice", age=30, email="alice@example.com")

# 转换为字典
user_dict = user.model_dump()
# {'name': 'Alice', 'age': 30, 'email': 'alice@example.com', 'is_active': True}

# 转换为 JSON 字符串
user_json = user.model_dump_json()

# 从字典创建
user2 = User.model_validate({"name": "Bob", "age": 25, "email": "bob@example.com"})
```

## 在 RAG 项目中的应用场景

1. **问答请求验证**：验证用户提交的问题格式（不能为空、长度限制等）
2. **文档上传验证**：验证上传的文档类型和大小
3. **响应格式化**：确保 API 返回的数据格式一致
4. **配置管理**：验证 RAG 系统的配置参数（API Key、模型名称等）

## 面试常问问题

### Q1: Pydantic 的作用是什么？为什么不用字典？

**答**：Pydantic 的核心作用是**数据验证**和**序列化**。相比字典，Pydantic 的优势：1）**类型验证**——自动检查字段类型，类型不对直接报错，字典不会；2）**必填/可选**——用 `...` 或 `Optional` 明确标注，字典需要手动判断；3）**范围约束**——`gt=0`、`max_length=100` 等，字典需要写大量 if 判断；4）**自动转换**——传入 `"30"` 会自动转成 `int 30`，字典不会；5）**IDE 支持**——有属性提示和自动补全，字典只有 `dict["key"]`；6）**自动生成文档**——FastAPI 根据 Pydantic 模型自动生成 Swagger 文档的请求/响应 Schema。总结：字典是"无约束的自由格式"，Pydantic 是"有约束的结构化数据"，在 API 场景下 Pydantic 更安全、更清晰。

### Q2: `Optional[str]` 和 `str` 有什么区别？

**答**：`str` 是必填字段，必须提供值；`Optional[str]` 等价于 `str | None`，字段可以为 `None` 或不提供。在 API 中，`str` 对应的请求体字段必须存在，`Optional[str]` 可以省略。

### Q3: Pydantic 的验证是在什么时候发生的？

**答**：Pydantic 的验证是在数据被赋值给模型实例时立即发生的。在 FastAPI 中，当请求到达时，FastAPI 会自动将请求数据传给 Pydantic 模型，如果验证失败，会自动返回 422 错误和详细的错误信息，处理函数根本不会被调用。
