# 依赖注入与异步

## 什么是依赖注入？

想象一个**公共设施**系统：每个家庭不需要自己打井、发电、修路，而是共享城市的供水、供电、道路系统。依赖注入也是同样的道理——把共享的功能（如数据库连接、配置读取）集中管理，需要的地方直接"注入"使用。

## 核心概念

### 1. Depends() 依赖注入

```python
from fastapi import Depends, FastAPI

app = FastAPI()

# 定义依赖项（公共设施）
def get_db():
    db = DatabaseConnection()  # 创建数据库连接
    try:
        yield db               # 提供给路由函数使用
    finally:
        db.close()             # 请求结束后关闭连接

# 在路由中使用依赖（使用公共设施）
@app.get("/users/")
async def get_users(db = Depends(get_db)):
    users = db.query("SELECT * FROM users")
    return users
```

类比：`Depends()` 就像水电管道的接口，路由函数通过这个接口获取共享资源。

### 2. 依赖注入的优势

1. **代码复用**：多个路由可以共享同一个依赖
2. **易于测试**：测试时可以用 mock 替换依赖
3. **关注分离**：业务逻辑和基础设施分离
4. **自动管理生命周期**：依赖的创建和清理由框架管理

### 3. 多层依赖

依赖可以嵌套：

```python
def get_config():
    return {"api_key": "xxx", "model": "gpt-3.5-turbo"}

def get_llm(config = Depends(get_config)):
    return ChatOpenAI(api_key=config["api_key"], model=config["model"])

@app.post("/ask")
async def ask(question: str, llm = Depends(get_llm)):
    response = llm.invoke(question)
    return {"answer": response.content}
```

### 4. async/await 异步编程

```python
import httpx

# 同步方式（阻塞）
def sync_request():
    response = httpx.get("https://api.example.com")  # 等待响应，期间不能做其他事
    return response.json()

# 异步方式（非阻塞）
async def async_request():
    async with httpx.AsyncClient() as client:
        response = await client.get("https://api.example.com")  # 等待期间可以处理其他请求
        return response.json()
```

类比：
- 同步：排队买奶茶，前面的人不走，你就一直等
- 异步：取号等奶茶，等待期间可以先去做别的事情

### 5. 异步的实际应用

```python
from fastapi import FastAPI
import httpx

app = FastAPI()

@app.get("/sync")                # 同步路由
def sync_endpoint():
    # 这里不能使用 await
    return {"message": "sync"}

@app.get("/async")               # 异步路由
async def async_endpoint():
    # 这里可以使用 await
    result = await some_async_operation()
    return {"message": "async", "result": result}

# 异步并发请求示例
@app.post("/batch-ask")
async def batch_ask(questions: list[str]):
    # 并发处理多个问题
    tasks = [process_question(q) for q in questions]
    results = await asyncio.gather(*tasks)
    return {"results": results}
```

### 6. 异步 vs 同步的选择

| 场景 | 推荐方式 | 原因 |
|------|----------|------|
| 调用外部 API | async | I/O 密集，异步更高效 |
| 数据库查询 | async | I/O 密集 |
| CPU 计算 | sync | CPU 密集，异步无优势 |
| 文件读写 | async | I/O 密集 |

## 在 RAG 项目中的应用场景

1. **共享 LLM 实例**：通过依赖注入共享 LLM 客户端，避免重复创建

## 面试常问问题

### Q1: 什么是依赖注入？FastAPI 的 Depends 有什么好处？

**答**：依赖注入是一种设计模式，将对象所依赖的外部资源从外部"注入"，而不是在内部创建。FastAPI 的 `Depends()` 好处：1）自动管理依赖的创建和销毁；2）支持多层依赖嵌套；3）便于单元测试（可以用 mock 替换）；4）代码复用，减少重复。

### Q2: async def 和 def 有什么区别？在 FastAPI 中怎么选？

**答**：`async def` 是异步函数，可以使用 `await`，适合 I/O 密集操作；`def` 是同步函数，适合 CPU 密集操作。在 FastAPI 中：如果函数内部需要 `await`（如调用外部 API），用 `async def`；如果不需要，用 `def`。FastAPI 会在独立的线程池中运行同步函数，不会阻塞主循环。

### Q3: 为什么 RAG API 推荐使用异步？

**答**：RAG 系统的主要操作是调用 LLM API 和向量数据库，这些都是 I/O 密集操作。使用异步可以让服务器在等待一个请求的 API 响应时，去处理其他请求，大大提高并发处理能力。例如，如果有 10 个用户同时提问，异步方式可以在等待第一个 LLM 响应时处理其他 9 个请求。
