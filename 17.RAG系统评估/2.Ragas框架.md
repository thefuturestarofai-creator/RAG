# Ragas 框架

## 核心概念

Ragas（Retrieval Augmented Generation Assessment）是一个专门用于评估 RAG 系统的开源框架。

### 类比理解

想象 RAG 系统是一个学生，Ragas 就是标准化考试系统：
- 出题（构造评估数据集）
- 考试（运行 RAG 系统）
- 评分（计算评估指标）
- 成绩单（生成评估报告）

## Ragas 核心组件

### 1. 评估数据集

Ragas 需要特定格式的评估数据集：

```python
{
    "question": "用户问题",
    "answer": "LLM 生成的回答",
    "contexts": ["检索到的文档1", "检索到的文档2", ...],
    "ground_truth": "标准答案（可选）"
}
```

**字段说明**：
- `question`：用户的问题
- `answer`：RAG 系统生成的回答
- `contexts`：检索到的上下文文档列表
- `ground_truth`：人工标注的标准答案（用于计算 Context Recall）

### 2. 评估指标

Ragas 内置了四个核心指标：

| 指标 | 需要 ground_truth | 评估维度 |
|------|------------------|---------|
| Faithfulness | 否 | 回答是否基于上下文 |
| Answer Relevancy | 否 | 回答是否切题 |
| Context Precision | 是 | 检索是否精确 |
| Context Recall | 是 | 检索是否全面 |

### 3. evaluate() 函数

核心评估函数：

```python
from ragas import evaluate

result = evaluate(
    dataset=eval_dataset,
    metrics=[faithfulness, answer_relevancy, context_precision, context_recall],
)
```

## 使用流程

### Step 1：安装 Ragas

```bash
pip install ragas
```

### Step 2：准备评估数据

```python
from datasets import Dataset

eval_data = {
    "question": ["什么是机器学习？", "Python 的优点是什么？"],
    "answer": ["机器学习是...", "Python 的优点是..."],
    "contexts": [["文档1", "文档2"], ["文档3", "文档4"]],
    "ground_truth": ["机器学习是...", "Python 的优点是..."],
}

dataset = Dataset.from_dict(eval_data)
```

### Step 3：配置 LLM

Ragas 使用 LLM 来做评估（需要语义理解能力）：

```python
from ragas.llms import LangchainLLMWrapper
from langchain_openai import ChatOpenAI

llm = LangchainLLMWrapper(ChatOpenAI(
    api_key="your-api-key",
    model="gpt-3.5-turbo"
))
```

### Step 4：运行评估

```python
from ragas import evaluate
from ragas.metrics import (
    faithfulness,
    answer_relevancy,
    context_precision,
    context_recall,
)

result = evaluate(
    dataset=dataset,
    metrics=[faithfulness, answer_relevancy, context_precision, context_recall],
    llm=llm,
)

print(result)
```

### Step 5：解读结果

```python
# 获取总体分数
print(f"总体分数: {result['faithfulness']}")

# 获取详细分数
for item in result.scores:
    print(f"问题: {item['question']}")
    print(f"  Faithfulness: {item['faithfulness']:.4f}")
    print(f"  Answer Relevancy: {item['answer_relevancy']:.4f}")
```

## 评估数据集构造

### 方法一：手动构造

适合小规模评估，质量最高：

```python
eval_data = {
    "question": "什么是 RAG？",
    "answer": "RAG 是检索增强生成...",
    "contexts": ["RAG 结合了检索和生成..."],
    "ground_truth": "RAG（Retrieval-Augmented Generation）是一种结合信息检索和文本生成的技术...",
}
```

### 方法二：从生产日志采样

从真实用户查询中采样：

```python
# 从日志中提取
logs = load_production_logs()

eval_data = {
    "question": [log["query"] for log in logs],
    "answer": [log["response"] for log in logs],
    "contexts": [log["retrieved_docs"] for log in logs],
}
```

### 方法三：LLM 辅助生成

用 LLM 生成评估数据：

```python
def generate_eval_data(documents, llm):
    """用 LLM 生成问题和标准答案"""
    eval_data = {"question": [], "ground_truth": []}

    for doc in documents:
        prompt = f"基于以下文档，生成一个相关问题和标准答案：\n{doc}"
        response = llm.generate(prompt)

        question = extract_question(response)
        answer = extract_answer(response)

        eval_data["question"].append(question)
        eval_data["ground_truth"].append(answer)

    return eval_data
```

## 结果解读

### 分数范围

所有指标分数范围都是 0-1：
- 0.8+：优秀
- 0.6-0.8：良好
- 0.4-0.6：一般
- 0.4 以下：需要优化

### 常见问题诊断

| 问题 | 可能原因 | 优化方向 |
|------|---------|---------|
| Faithfulness 低 | LLM 幻觉严重 | Prompt 约束、Temperature=0 |
| Answer Relevancy 低 | 回答跑题 | 优化 Prompt、限制长度 |
| Context Precision 低 | 检索噪声多 | Re-ranking、提高阈值 |
| Context Recall 低 | 检索遗漏 | 混合检索、Query 改写 |

## 面试加分话术

### 问题：你用什么框架评估 RAG？

**回答**：
"我主要用 Ragas 框架。它专门针对 RAG 系统设计，内置了四个核心指标：Faithfulness、Answer Relevancy、Context Precision、Context Recall。使用流程是：准备评估数据集、配置 LLM、运行 evaluate() 函数、解读结果。Ragas 的优势是自动化程度高，可以用 LLM 做语义级别的评估。"

### 问题：评估数据集怎么构造？

**回答**：
"我通常用三种方式结合：
1. 手动构造：核心 case，质量最高
2. 生产日志采样：真实场景，代表性好
3. LLM 辅助生成：大规模扩展

关键是保证数据多样性，覆盖不同类型的问题（简单/复杂、事实/推理）。"

### 问题：评估结果不理想怎么办？

**回答**：
"我会按优先级排查：
1. 先看 Faithfulness，如果低说明幻觉严重，需要加 Prompt 约束
2. 再看 Context Recall，如果低说明检索遗漏，需要优化检索
3. 然后看 Context Precision，如果低说明噪声多，需要加 Re-ranking
4. 最后看 Answer Relevancy，如果低说明回答跑题，需要优化 Prompt

这个顺序是因为 Faithfulness 是 RAG 的核心价值。"

## 常见追问

**Q: Ragas 的评估准确吗？**

A: Ragas 用 LLM 做评估，有一定主观性，但比人工评估更一致、可重复。建议结合人工评估做校验。

**Q: 评估成本高吗？**

A: 每个样本大约需要 3-5 次 LLM 调用。100 个样本用 GPT-3.5 大约 $1-2。可以用小模型降低成本。

**Q: 除了 Ragas 还有什么评估框架？**

A: 
- TruLens：可视化评估
- DeepEval：更多评估指标
- LangSmith：LangChain 官方评估工具
- 自定义：根据业务需求定制

**Q: 评估频率怎么定？**

A: 
- 开发阶段：每次改动都评估
- 上线前：完整评估
- 生产环境：定期抽样评估（如每周）
