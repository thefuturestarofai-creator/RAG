# RAG 评估指标

## 核心概念

RAG 评估就像"考试评分"——需要从多个维度评估系统的回答质量。

### 类比理解

想象老师批改学生的问答作业：
- **Context Precision（上下文精确率）**：学生引用的参考资料是否相关？
- **Context Recall（上下文召回率）**：学生是否找到了所有相关资料？
- **Faithfulness（忠实度）**：学生的回答是否基于参考资料，有没有编造？
- **Answer Relevancy（答案相关性）**：学生的回答是否回答了问题？

## 四大评估指标详解

### 1. Context Precision（上下文精确率）

**定义**：检索到的文档中，有多少是真正相关的？

**公式**：
```
Context Precision = 相关文档数 / 检索到的文档总数
```

**类比**：在图书馆借了 10 本书，其中 7 本是真正需要的，精确率 = 70%

**意义**：
- 衡量检索的"精准度"
- 高精确率意味着检索结果噪声少
- 影响 LLM 回答的质量（噪声多会干扰）

**优化方向**：
- 优化 embedding 模型
- 调整相似度阈值
- 使用 Re-ranking

### 2. Context Recall（上下文召回率）

**定义**：所有相关文档中，有多少被检索到了？

**公式**：
```
Context Recall = 检索到的相关文档数 / 所有相关文档总数
```

**类比**：图书馆有 20 本相关书，你找到了 15 本，召回率 = 75%

**意义**：
- 衡量检索的"全面性"
- 高召回率意味着不遗漏重要信息
- 影响回答的完整性

**优化方向**：
- 增加检索数量（Top-K）
- 使用混合检索
- Query 改写和扩展

### 3. Faithfulness（忠实度）

**定义**：LLM 的回答是否忠于检索到的上下文？

**评估方法**：
1. 将回答分解成多个陈述
2. 检查每个陈述是否有上下文支持
3. 计算有支持的陈述比例

**示例**：
```
上下文："Python 是一种高级编程语言，由 Guido van Rossum 创建"
回答："Python 是一种高级编程语言，由 Guido van Rossum 在 1991 年创建"

检查：
- "Python 是一种高级编程语言" → 有支持 ✓
- "由 Guido van Rossum 创建" → 有支持 ✓
- "在 1991 年创建" → 无支持 ✗（上下文没提年份）

Faithfulness = 2/3 ≈ 67%
```

**意义**：
- 衡量回答的"可靠性"
- 高忠实度意味着减少幻觉
- 是 RAG 系统的核心优势

**优化方向**：
- Prompt 约束（"只基于提供的上下文回答"）
- Temperature 设为 0
- 引用来源

### 4. Answer Relevancy（答案相关性）

**定义**：回答是否与问题相关？

**评估方法**：
1. 从回答中生成可能的问题
2. 计算生成问题与原始问题的相似度
3. 相似度越高，相关性越高

**示例**：
```
问题："Python 的优点是什么？"
回答："Python 语法简洁，适合初学者，有丰富的库支持"

生成的问题：
- "Python 有什么特点？"
- "为什么 Python 适合初学者？"

与原问题的相似度 → Answer Relevancy
```

**意义**：
- 衡量回答的"针对性"
- 高相关性意味着回答切题
- 避免答非所问

**优化方向**：
- 优化 Prompt（明确问题）
- 限制回答长度
- 使用 CoT（思维链）

## 指标之间的关系

### 权衡取舍

```
高召回率 ←→ 高精确率
    ↓           ↓
更多上下文   更少噪声
    ↓           ↓
可能有噪声   可能遗漏
```

### 综合评估

| 场景 | 优先指标 | 原因 |
|------|---------|------|
| 客服系统 | Faithfulness | 避免错误信息 |
| 知识库搜索 | Context Recall | 不遗漏信息 |
| 精准问答 | Context Precision | 减少干扰 |
| 开放对话 | Answer Relevancy | 回答切题 |

## 面试加分话术

### 问题：RAG 系统怎么评估？

**回答**：
"RAG 评估主要看四个维度：
1. Context Precision：检索的精确率，相关文档占比
2. Context Recall：检索的召回率，是否找全
3. Faithfulness：忠实度，回答是否基于上下文
4. Answer Relevancy：答案相关性，是否切题

实践中，我会用 Ragas 框架做自动化评估，同时结合人工评估。重点关注 Faithfulness，因为这是 RAG 相比纯 LLM 的核心优势。"

### 问题：怎么平衡精确率和召回率？

**回答**：
"这取决于业务场景。客服系统优先精确率，避免错误信息；知识库搜索优先召回率，不遗漏信息。实践中，我会用混合检索 + Re-ranking 来同时优化两者。先用召回率高的方法获取候选，再用精确率高的方法筛选。"

### 问题：Faithfulness 怎么计算？

**回答**：
"Faithfulness 的计算分三步：
1. 将 LLM 回答分解成多个独立陈述
2. 对每个陈述，检查是否有上下文支持
3. 计算有支持的陈述比例

这通常用 LLM 来做，因为需要语义理解。Ragas 框架内置了这个功能。"

## 常见追问

**Q: 为什么不直接用准确率（Accuracy）？**

A: RAG 是开放域问答，没有标准答案。准确率需要明确的对错判断，而 RAG 评估更关注质量维度（相关性、忠实度等）。

**Q: 这些指标怎么自动化计算？**

A: 使用 Ragas、TruLens 等评估框架。它们用 LLM 来做评估，因为需要语义理解能力。

**Q: 人工评估和自动评估怎么结合？**

A: 
1. 自动评估：大规模、快速、可重复
2. 人工评估：小规模、准确、有深度
3. 结合方式：自动评估筛选问题 case，人工评估深入分析

**Q: 评估数据集怎么构造？**

A: 
1. 从真实用户查询中采样
2. 人工标注相关文档和标准答案
3. 使用 LLM 辅助生成
4. 注意数据多样性
