# 向量存储与检索

## 什么是向量存储与检索？

想象一个**智能书架**：你把每本书（文档）放上去时，书架会自动理解书的内容，并按照内容的相似性来排列。当你问一个问题时，书架能快速找到与你问题最相关的几本书。

这就是向量存储与检索系统做的事情。

## 核心概念

### 1. VectorStore（向量数据库）

VectorStore 是专门存储和检索向量的数据库。LangChain 支持多种向量数据库：

| 向量数据库 | 特点 | 适用场景 |
|------------|------|----------|
| Chroma | 轻量级、本地运行 | 开发测试、小规模应用 |
| FAISS | Facebook开源、高性能 | 大规模检索 |
| Pinecone | 云托管、易扩展 | 生产环境 |
| Milvus | 开源、支持分布式 | 大规模生产环境 |
| Weaviate | 支持混合搜索 | 需要多种搜索方式 |

### 2. Chroma 集成

Chroma 是 RAG 学习中最推荐的向量数据库，因为它：
- 安装简单（`pip install chromadb`）
- 无需外部服务，本地运行
- 与 LangChain 无缝集成

```python
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings

# 创建向量数据库
vectorstore = Chroma.from_documents(
    documents=chunks,                    # 切分后的文档
    embedding=OpenAIEmbeddings(          # Embedding 模型
        api_key="your-api-key",
        base_url="https://api.openai.com/v1"
    ),
    persist_directory="./chroma_db"      # 持久化目录
)
```

### 3. Retriever（检索器）

Retriever 是从 VectorStore 中检索相关文档的接口：

```python
# 将 VectorStore 转换为 Retriever
retriever = vectorstore.as_retriever(
    search_type="similarity",    # 检索类型
    search_kwargs={"k": 3}       # 返回前3个最相似的文档
)

# 检索
docs = retriever.invoke("什么是RAG？")
for doc in docs:
    print(doc.page_content)
```

#### 检索类型

| 类型 | 说明 |
|------|------|
| `similarity` | 基于向量相似度检索（默认） |
| `mmr` | 最大边际相关性（增加结果多样性） |
| `similarity_score_threshold` | 带相似度阈值的检索 |

### 4. 完整的 RAG 流程

```python
# 1. 加载文档
loader = TextLoader("knowledge.txt")
documents = loader.load()

# 2. 切分文档
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(documents)

# 3. 创建向量数据库
vectorstore = Chroma.from_documents(chunks, embedding=embeddings)

# 4. 创建检索器
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

# 5. 检索相关文档
docs = retriever.invoke("用户的问题")

# 6. 将检索结果交给 LLM 生成答案
```

## 在 RAG 项目中的应用场景

1. **知识库问答**：将文档存储到向量数据库，用户提问时检索相关文档，LLM 基于检索结果生成答案
2. **文档推荐**：根据用户当前阅读的文档，推荐相似的文档
3. **语义去重**：利用向量相似度检测和去除重复文档
4. **多源检索**：从多个向量数据库中检索，合并结果

## 面试常问问题

### Q1: VectorStore 和传统数据库有什么区别？

**答**：传统数据库（如 MySQL）存储结构化数据，通过精确匹配（WHERE name = 'xxx'）查询。VectorStore 存储向量数据，通过相似度检索，找到语义最接近的结果。传统数据库适合精确查询，VectorStore 适合语义搜索。两者在 RAG 中通常配合使用——VectorStore 做语义检索，传统数据库存储元数据。

### Q2: 检索时 k 值设为多少合适？

**答**：k 值取决于应用场景。一般建议：
- 简单问答：k=3-5，找到最相关的几个片段即可
- 复杂分析：k=5-10，需要更多上下文
- k 太大会引入噪声（不相关的内容），k 太小可能遗漏重要信息
- 实际项目中需要根据效果调优

### Q3: Chroma 和 FAISS 有什么区别？怎么选？

**答**：Chroma 是一个完整的向量数据库，支持持久化、元数据过滤、CRUD 操作，适合开发和小规模应用。FAISS 是 Facebook 的相似度搜索库，性能更高，但更像是一个"搜索引擎引擎"，需要自己管理数据。选择建议：学习和原型开发用 Chroma，大规模生产环境考虑 FAISS 或 Milvus。
