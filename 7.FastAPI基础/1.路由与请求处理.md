# 路由与请求处理

## 什么是 FastAPI？

FastAPI 是一个现代、高性能的 Python Web 框架，专门用于构建 API。它的特点是：
- 速度快（与 Node.js、Go 相当）
- 自动生成 API 文档
- 类型提示支持
- 简洁易学

类比：如果把 Web 框架比作餐厅，FastAPI 就是一个高效的智能餐厅，菜单（路由）清晰，服务员（处理函数）专业，还能自动生成菜单说明（API文档）。

## 核心概念

### 1. 路由（Route）

路由就是 URL 和处理函数的映射关系：

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")           # GET 请求，路径为 /
async def root():
    return {"message": "Hello World"}

@app.get("/items/{item_id}")  # GET 请求，路径带参数
async def read_item(item_id: int):
    return {"item_id": item_id}
```

类比：路由就像餐厅菜单上的菜品编号，每个编号对应一道菜（处理函数）。

### 2. HTTP 方法

| 方法 | 装饰器 | 用途 | 类比 |
|------|--------|------|------|
| GET | `@app.get()` | 获取数据 | 看菜单 |
| POST | `@app.post()` | 创建数据 | 点菜 |
| PUT | `@app.put()` | 更新数据 | 换菜 |
| DELETE | `@app.delete()` | 删除数据 | 退菜 |

### 3. 路径参数（Path Parameter）

URL 中的变量部分：

```python
@app.get("/users/{user_id}")
async def get_user(user_id: int):    # 类型自动转换和验证
    return {"user_id": user_id}

# 访问 /users/123 → {"user_id": 123}
```

### 4. 查询参数（Query Parameter）

URL 中 `?` 后面的参数：

```python
@app.get("/items/")
async def list_items(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

# 访问 /items/?skip=20&limit=5 → {"skip": 20, "limit": 5}
```

### 5. 请求体（Request Body）

POST 请求中发送的数据：

```python
from pydantic import BaseModel

class Item(BaseModel):
    name: str
    price: float
    is_offer: bool = False

@app.post("/items/")
async def create_item(item: Item):
    return {"item_name": item.name, "item_price": item.price}
```

### 6. 自动文档 Swagger UI

FastAPI 最强大的特性之一：**自动生成交互式 API 文档**。

启动 FastAPI 应用后，访问以下路径即可看到自动生成的文档：
- `/docs` — Swagger UI（推荐，支持在线测试接口）
- `/redoc` — ReDoc（更美观的文档展示）

```python
# 无需任何额外代码，FastAPI 自动根据路由和 Pydantic 模型生成文档
app = FastAPI(
    title="我的 RAG API",
    description="基于检索增强生成的问答系统",
    version="1.0.0"
)
```

Swagger UI 的好处：
1. **在线测试**：直接在浏览器中调用 API，无需 Postman
2. **自动生成**：代码即文档，永远和代码同步
3. **团队协作**：前端开发者可以直接查看接口定义
4. **请求/响应 Schema**：自动展示 Pydantic 模型的字段说明和验证规则

## 在 RAG 项目中的应用场景

1. **问答接口**：`POST /ask` 接收用户问题，返回 RAG 生成的答案
2. **文档管理接口**：`POST /documents` 上传文档，`GET /documents` 查看文档列表
3. **检索接口**：`POST /search` 执行语义检索
4. **健康检查**：`GET /health` 检查服务状态

## 面试常问问题

### Q1: FastAPI 和 Flask 有什么区别？

**答**：主要区别：1）FastAPI 基于 ASGI，支持异步；Flask 基于 WSGI，同步为主。2）FastAPI 使用 Python 类型提示，自动生成 API 文档；Flask 需要额外工具。3）FastAPI 性能更高，接近 Node.js。4）FastAPI 内置数据验证（Pydantic）；Flask 需要手动处理。5）FastAPI 更适合构建 API，Flask 更通用。

### Q2: 路径参数和查询参数有什么区别？

**答**：路径参数是 URL 路径的一部分（如 `/users/123` 中的 `123`），用于标识资源。查询参数在 URL 的 `?` 后面（如 `/items?skip=10`），用于过滤、分页等可选配置。路径参数通常是必需的，查询参数通常有默认值。

### Q3: 为什么 FastAPI 的处理函数用 `async def`？

**答**：`async def` 声明函数为异步函数，可以使用 `await` 等待 I/O 操作（如调用 API、查询数据库），在等待期间可以处理其他请求，提高并发性能。对于 RAG 应用，调用 LLM API 是 I/O 密集操作，使用异步可以显著提高吞吐量。
