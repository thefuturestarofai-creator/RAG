# Gradio 备选方案

## 一句话理解

如果 Streamlit 是"瑞士军刀"，那 Gradio 就是"手术刀"——更专注、更精准，专门为 ML 模型演示而设计。做聊天界面，Gradio 可能更顺手。

---

## 一、Gradio 核心概念

### 1. Interface — 最简方式

```python
import gradio as gr

def greet(name):
    return f"你好，{name}！"

# 最简单的界面：一个输入框 + 一个输出框
demo = gr.Interface(
    fn=greet,              # 处理函数
    inputs="text",         # 输入类型
    outputs="text",        # 输出类型
    title="打招呼程序"
)

demo.launch()
```

### 2. ChatInterface — 聊天专用

```python

        history: 聊天历史 [(用户消息, AI回复), ...]
    """
    # 调用 RAG API
    answer = call_rag_api(message)
    return answer

demo = gr.ChatInterface(
```

### 3. Blocks — 自定义布局

```python
import gradio as gr

with gr.Blocks(title="RAG 系统") as demo:
    gr.Markdown("# RAG 知识库问答系统")

    with gr.Row():
        with gr.Column(scale=3):
            # 聊天区域
            chatbot = gr.Chatbot(height=400)
            msg = gr.Textbox(placeholder="请输入问题...")
        with gr.Column(scale=1):
            # 设置区域
            gr.Markdown("## 设置")
            model = gr.Dropdown(["GPT-3.5", "GPT-4"], value="GPT-3.5")
            top_k = gr.Slider(1, 10, value=3, label="检索数量")
            clear = gr.Button("清空对话")

    # 绑定事件
    def respond(message, chat_history, model, top_k):
        answer = call_rag_api(message, model, top_k)
        chat_history.append((message, answer))
        return "", chat_history

    msg.submit(respond, [msg, chatbot, model, top_k], [msg, chatbot])
    clear.click(lambda: [], None, chatbot)

demo.launch()
```

## 二、Streamlit vs Gradio 对比

| 特性 | Streamlit | Gradio |
|------|-----------|--------|
| 学习曲线 | 低 | 低 |
| 聊天界面 | st.chat_message | ChatInterface（更简洁） |
| 自定义布局 | 灵活但略复杂 | Blocks 系统很强大 |
| 流式输出 | 需要手动实现 | 原生支持 |
| 部署 | Streamlit Cloud | Hugging Face Spaces |
| 适用场景 | 通用 Web 应用 | ML 模型演示 |

**选择建议**：
- 如果你只需要一个聊天界面 → Gradio（更简单）
- 如果你需要更复杂的布局和功能 → Streamlit（更灵活）
- 如果你要部署到 Hugging Face → Gradio（原生支持）

## 六、面试常问问题

### Q1: Gradio 的 ChatInterface 怎么用？
**答**：传入一个处理函数 fn，接收 message（当前消息）和 history（历史记录），返回回答字符串。ChatInterface 自动处理聊天界面的渲染、历史记录管理等。

### Q2: Gradio 和 Streamlit 怎么选？
**答**：如果只需要聊天界面，选 Gradio（ChatInterface 更简洁）；如果需要复杂的布局和多种组件，选 Streamlit（更灵活）。两者都可以做 RAG 前端。

### Q3: Gradio 怎么实现流式输出？
**答**：处理函数用 yield 代替 return，每次 yield 一个中间结果，Gradio 会自动更新界面。比如 `for chunk in llm.stream(): response += chunk; yield response`。

### Q4: Gradio 的 Blocks 是什么？
**答**：Blocks 是 Gradio 的底层 API，提供完全自定义的布局能力。可以用 Row、Column、Tab 等组件组织界面，比 Interface 和 ChatInterface 更灵活。

### Q5: 如何把 Gradio 应用部署到线上？
**答**：最简单的方式是部署到 Hugging Face Spaces——上传 app.py 和 requirements.txt，自动构建和部署。也可以用 Docker 部署到自己的服务器。
