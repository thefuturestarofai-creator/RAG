# Streamlit 快速上手

## 一句话理解

Streamlit 就像"前端界的乐高"——你不需要学 HTML/CSS/JavaScript，只用 Python 代码就能拼出一个漂亮的 Web 界面。它是做 AI 原型演示的最佳选择。

---

## 一、为什么选 Streamlit？

| 方案 | 学习曲线 | 开发速度 | 适用场景 |
|------|---------|---------|---------|
| React/Vue | 高 | 慢 | 正式产品 |
| Streamlit | 低 | 极快 | 原型演示、内部工具 |
| Gradio | 低 | 快 | ML 模型演示 |

**类比**：如果 React 是"专业摄影器材"，那 Streamlit 就是"手机相机"——功能没那么全，但拍个朋友圈绰绰有余。

---

## 二、核心组件

### 1. 标题和文本

```python
import streamlit as st

st.title("RAG 知识库问答系统")      # 大标题
st.header("功能介绍")               # 中标题
st.subheader("检索增强生成")         # 小标题
st.text("这是一段普通文本")          # 纯文本
st.markdown("**加粗** *斜体*")      # Markdown 格式
st.write("可以写任何内容")           # 万能输出
```

### 2. 用户输入

```python
# 文本输入框
question = st.text_input("请输入你的问题", placeholder="什么是RAG？")

# 文本区域（多行）
content = st.text_area("请输入文档内容", height=200)

# 按钮
if st.button("提交"):
    st.write(f"你的问题是：{question}")

# 下拉选择
model = st.selectbox("选择模型", ["GPT-3.5", "GPT-4"])

# 滑块
top_k = st.slider("检索数量", min_value=1, max_value=10, value=3)

# 文件上传
uploaded_file = st.file_uploader("上传文档", type=["txt", "pdf", "md"])
```

### 3. 显示结果

```python
# 普通文本
st.write("回答内容")

# 代码块
st.code("print('Hello')", language="python")

# JSON
st.json({"answer": "RAG是...", "sources": ["doc1.txt"]})

# 表格
import pandas as pd
df = pd.DataFrame({"来源": ["doc1.txt", "doc2.txt"], "相关度": [0.95, 0.87]})
st.table(df)

# 进度条
progress = st.progress(0)
for i in range(100):
    progress.progress(i + 1)
```

### 4. 布局

```python
# 侧边栏
with st.sidebar:
    st.title("设置")
    api_key = st.text_input("API Key", type="password")
    model = st.selectbox("模型", ["GPT-3.5", "GPT-4"])

# 分栏
col1, col2 = st.columns(2)
with col1:
    st.write("左栏内容")
with col2:
    st.write("右栏内容")

# 标签页
tab1, tab2 = st.tabs(["问答", "知识库"])
with tab1:
    st.write("问答内容")
with tab2:
    st.write("知识库管理")
```

## 三、聊天界面

Streamlit 提供了专门的聊天组件：

```python
import streamlit as st

st.title("RAG 聊天助手")

# 初始化聊天历史
if "messages" not in st.session_state:
    st.session_state.messages = []

# 显示历史消息
for message in st.session_state.messages:
    with st.chat_message(message["role"]):
        st.write(message["content"])

# 用户输入
if prompt := st.chat_input("请输入问题"):
    # 显示用户消息
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.write(prompt)

    # 生成回答
    with st.chat_message("assistant"):
        with st.spinner("思考中..."):
            answer = call_rag_api(prompt)  # 调用 RAG API
            st.write(answer)

    # 保存回答
    st.session_state.messages.append({"role": "assistant", "content": answer})
```

## 四、session_state 状态管理

```python
import streamlit as st

# session_state 像一个"记忆"，在用户会话期间保持状态
if "count" not in st.session_state:
    st.session_state.count = 0

if st.button("点击 +1"):
    st.session_state.count += 1

st.write(f"点击次数: {st.session_state.count}")
```

**类比**：session_state 像浏览器的 localStorage——同一个用户刷新页面后，数据还在。

## 六、面试常问问题

### Q1: Streamlit 是什么？适合什么场景？
**答**：Streamlit 是一个用 Python 快速构建 Web 应用的框架，不需要前端知识。适合 AI/ML 原型演示、内部工具、数据看板等场景。不适合需要复杂交互或高并发的正式产品。

### Q2: Streamlit 的 session_state 是什么？
**答**：session_state 是 Streamlit 的会话状态管理机制。每个用户的浏览器会话有一个独立的 session_state，可以存储变量（如聊天历史、用户设置），刷新页面后数据还在。

### Q3: Streamlit 和 Gradio 有什么区别？
**答**：Streamlit 更通用，适合各种 Web 应用；Gradio 更专注于 ML 模型的演示，提供了更多的 ML 专用组件（如图像上传、音频播放）。对于 RAG 聊天界面，两者都可以。

### Q4: Streamlit 的性能怎么样？
**答**：Streamlit 每次交互都会重新运行整个脚本，对于简单应用没问题，但对于复杂应用可能较慢。可以通过 session_state 缓存数据、用 st.cache_data 缓存计算结果来优化。

### Q5: 如何在 Streamlit 中调用后端 API？
**答**：用 requests 库调用后端的 HTTP 接口。对于流式输出，可以用 requests 的 stream 参数或 SSE 客户端库。
