# Docker 实战

## Dockerfile 编写

### 常用指令

| 指令 | 作用 | 示例 |
|------|------|------|
| `FROM` | 指定基础镜像 | `FROM python:3.11-slim` |
| `WORKDIR` | 设置工作目录 | `WORKDIR /app` |
| `COPY` | 复制文件到镜像 | `COPY . .` |
| `RUN` | 构建时执行命令 | `RUN pip install -r requirements.txt` |
| `EXPOSE` | 声明端口 | `EXPOSE 8000` |
| `CMD` | 容器启动命令 | `CMD ["uvicorn", "main:app"]` |
| `ENV` | 设置环境变量 | `ENV PYTHONUNBUFFERED=1` |

### FastAPI 项目 Dockerfile 示例

```dockerfile
# 使用 Python 轻量镜像
FROM python:3.11-slim

# 设置工作目录
WORKDIR /app

# 复制依赖文件（利用 Docker 缓存）
COPY requirements.txt .

# 安装依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动应用
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 构建和运行

```bash
# 构建镜像
docker build -t my-rag-app .

# 运行容器
docker run -d -p 8000:8000 --name rag-app my-rag-app

# 查看日志
docker logs rag-app

# 进入容器
docker exec -it rag-app bash
```

## Docker Compose

### docker-compose.yml 结构

```yaml
version: '3.8'

services:
  # 服务1：FastAPI 应用
  api:
    build: .                          # 从 Dockerfile 构建
    ports:
      - "8000:8000"                   # 端口映射
    environment:
      - API_KEY=your-api-key          # 环境变量
    volumes:
      - ./data:/app/data              # 数据卷
    depends_on:
      - chroma                        # 依赖关系
    restart: always                   # 自动重启

  # 服务2：Chroma 向量数据库
  chroma:
    image: chromadb/chroma:latest     # 使用官方镜像
    ports:
      - "8001:8000"
    volumes:
      - chroma_data:/chroma/chroma    # 持久化数据
    restart: always

# 定义命名卷
volumes:
  chroma_data:
```

### 常用命令

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f api

# 停止所有服务
docker-compose down

# 重新构建并启动
docker-compose up -d --build
```

## 端口映射

端口映射格式：`宿主机端口:容器端口`

```bash
# 将容器的 8000 端口映射到宿主机的 8000 端口
docker run -p 8000:8000 my-app

# 将容器的 8000 端口映射到宿主机的 9000 端口
docker run -p 9000:8000 my-app
```

## 数据卷（Volume）

数据卷用于持久化容器中的数据：

```bash
# 命名卷（推荐）
docker run -v my_data:/app/data my-app

# 绑定挂载（开发环境常用）
docker run -v $(pwd):/app my-app
```

## 在 RAG 项目中的应用场景

1. **一键部署**：用 Dockerfile 打包 RAG 应用，一条命令部署到任何服务器
2. **服务编排**：用 Docker Compose 同时启动 FastAPI 服务和 Chroma 数据库
3. **环境隔离**：开发、测试、生产环境使用不同的 Docker 配置
4. **水平扩展**：通过 `docker-compose up --scale api=3` 快速扩展服务实例

## 面试常问问题

### Q1: COPY 和 ADD 有什么区别？

**答**：`COPY` 只做简单的文件复制。`ADD` 除了复制文件，还支持：1）自动解压 tar 文件。2）从 URL 下载文件。一般建议优先使用 `COPY`，因为它的行为更明确、更可预测。只有在需要自动解压时才用 `ADD`。

### Q2: 为什么 Dockerfile 中要先 COPY requirements.txt 再 COPY 代码？

**答**：这是利用 Docker 的层缓存机制。Docker 会缓存每一层的结果，如果某一层的输入没有变化，就使用缓存。先复制 requirements.txt 并安装依赖，如果依赖没变，这一层会使用缓存。之后复制代码时，只有代码层需要重新构建。这样可以大大加快构建速度。

### Q3: docker-compose.yml 中的 depends_on 有什么作用？

**答**：`depends_on` 指定服务之间的依赖关系。它确保被依赖的服务先启动。例如 `depends_on: - chroma` 表示在启动 API 服务之前，先启动 Chroma 服务。但注意：`depends_on` 只保证启动顺序，不保证服务完全就绪。如果应用需要等待数据库就绪，需要在代码中实现重试机制。
