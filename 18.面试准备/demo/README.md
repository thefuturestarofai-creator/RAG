## 常见问题

### Q1: 面试时紧张怎么办？

A: 
1. 充分准备：熟悉所有知识点和项目细节
2. 模拟练习：多做模拟面试，习惯口头表达
3. 结构化回答：用框架组织思路，减少紧张
4. 深呼吸：面试前做几次深呼吸，放松心情

### Q2: 被问到不会的问题怎么办？

A: 
1. 诚实承认："这个问题我不太了解"
2. 展示思考："但我可以这样思考..."
3. 关联已知："我知道的相关知识是..."
4. 学习态度："我回去会深入学习"

### Q3: 如何展示技术深度？

A: 
1. 讲原理：不只说"用了什么"，要说"为什么用"
2. 讲对比：说明考虑过哪些方案，为什么选这个
3. 讲优化：展示问题-方案-效果的完整思路
4. 讲反思：展示自我改进的能力

### Q4: 项目讲解超时怎么办？

A: 
1. 准备精简版：每个部分准备 1 分钟版本
2. 识别重点：难点优化是核心，可以多花时间
3. 灵活调整：根据面试官反应调整节奏
4. 主动控制："这部分我就简单讲一下"

1. 什么是 RAG？为什么需要 RAG 而不是直接用 LLM？
2. Embedding 是什么？和关键词匹配有什么区别？

4. 向量检索的原理是什么？HNSW 大致怎么工作？
5. 你用了什么检索策略？有没有做优化？
6. FastAPI 为什么选异步？和 Flask 区别？
7. Docker 解决了什么问题？镜像和容器的关系？
8. 你怎么做 RAG 系统的评估？
9. 遇到过幻觉问题吗？怎么处理的？
