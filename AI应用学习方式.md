# AI 应用学习方式

> 本文档定义了学习流程和文件组织规范，请在每次学习开始时读取本文档。

> 使用方法推荐:读取 学习路线.md,AI应用学习方式.md，学习进度.md,开始学习
---

## 一、学习流程（5 步循环）

每个章节按以下步骤推进：

### 第 1 步：生成理论文档
- 根据学习路线上的章节，生成该章节的技术讲解文件（.md 格式）
- 每个知识点一个独立的 .md 文件

### 第 2 步：生成 Demo 代码
- 为每个理论知识点创建对应的可运行 Demo（.py 文件）
- 所有 Demo 放在章节目录下的 `demo/` 文件夹中
- 包含一个 `README.md`，说明学习顺序、每个 Demo 的作用、与理论文档的对应关系

### 第 3 步：学习理论知识
- 阅读理论文档，遇到不理解的内容直接问 AI
- AI 讲解后，让AI将讲解内容补充到对应的理论文档旁边（不是另起一个文件，而是加到原文对应位置）

### 第 4 步：实操 Demo
- 阅读 Demo 代码，遇到不理解的代码直接问 AI
- AI 讲解后，让AI将讲解以注释形式写到代码旁边
- 理解后运行 Demo，观察输出结果

### 第 5 步：更新进度
- 本章节学习完成后，告诉 AI"更新学习进度"
- AI 将本章节的学习内容、核心理解、产出文件记录到 `学习进度.md`
- 下次学习时，先让 AI 读取 `学习进度.md`，然后说"开始学习"即可继续

## 二、章节文件结构规范

每个章节的目录结构如下：

```
N.章节名称/
├── 1.知识点A.md          # 理论讲解
├── 2.知识点B.md          # 理论讲解
├── 3.知识点C.md          # 理论讲解
└── demo/
    ├── README.md          # 学习顺序 + Demo说明 + 理论对应关系
    ├── 1_demo_a.py        # 知识点A的操作实例
    ├── 2_demo_b.py        # 知识点B的操作实例
    └── 3_demo_c.py        # 知识点C的操作实例
```

### README.md 格式规范

每个 Demo 按以下格式描述：

```markdown
### Demo N: 标题（一句话定位）
**文件**: `文件名.py`

这个 Demo 做什么、演示什么核心概念（2-3 句话）

**需要 API Key / 不需要 API Key**
```

末尾包含：
- **对应理论文档**：表格列出每个 Demo 与理论 .md 文件的映射关系
- **核心要点**：总结本章节的关键概念

---

## 三、关键约定

| 约定 | 说明 |
|------|------|
| 理论讲解加到原文旁边 | 不新建文件，直接补充到对应知识点的位置 |
| 代码讲解写成注释 | 不新建文件，直接注释到代码对应行的旁边 |
| Demo API 配置 | 代码中统一使用变量 `API_KEY`、`BASE_URL`、`MODEL`，学习者自行填写自己的模型配置 |
| 进度记录在 1.学习进度.md | 每章学完后更新，包含学习内容、核心理解、产出文件 |
| 学习路线在 1.学习路线.md | 章节顺序以此为准 |
