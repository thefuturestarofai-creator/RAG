## Demo 列表

### Demo 1: 代码重构前后对比
**文件**: `1_demo_代码重构前后对比.py`

用一个真实的 RAG 文档处理场景，展示"重构前"（重复代码、职责混乱、无类型注解）和"重构后"（DRY、单一职责、类型注解、异常处理）的代码差异。最后附带一个重构检查清单。

**不需要 API Key**

---

### Demo 2: 项目文档生成
**文件**: `2_demo_项目文档生成.py`

演示如何为 RAG 项目生成规范的文档：README.md（项目说明）、CHANGELOG.md（变更日志）、架构图（ASCII + Mermaid 两种格式）。生成的文档可以直接复制到项目中使用。

**不需要 API Key**

## 核心要点

学完本章后，你应该能够：

1. **识别代码坏味道** — 重复代码、职责混乱、命名不清、缺少类型注解
2. **应用重构原则** — DRY、单一职责、模块化设计
3. **撰写规范文档** — README（快速开始）、CHANGELOG（版本记录）、架构图
4. **面试加分** — 能说出"我在项目中做了代码重构，提高了可维护性"
