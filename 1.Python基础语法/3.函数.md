# 函数

> 函数就是"把一段代码打包，起个名字，随时调用"。RAG 系统中，`load_document()`、`search()`、`generate_answer()` 这些都是函数——每个函数负责一件事，组合起来就是完整的 RAG 管道。

---

## 1. 基本语法

```python
def greet(name):
    """向某人打招呼（这是文档字符串，说明函数做什么）"""
    print(f"你好，{name}！")

# 调用函数
greet("RAG学习者")    # 你好，RAG学习者！
```

**关键要素：**
- `def` — 声明"这是一个函数"
- 函数名 — 推荐 snake_case，如 `load_document`
- `()` — 参数列表
- `:` — 冒号不能少
- 函数体 — 缩进的代码块
- `"""文档字符串"""` — 说明函数做什么（可选但强烈推荐）

## 2. 参数：函数的"输入"

### 2.1 位置参数

```python
def split_text(text, chunk_size):
    """把文本按指定大小切分"""
    return [text[i:i+chunk_size] for i in range(0, len(text), chunk_size)]

chunks = split_text("这是一段很长的文本内容", 5)
# ['这是一段很长', '的文本内容']
```

### 2.2 默认参数

```python
def search(query, top_k=3, threshold=0.5):
    """搜索相关文档，top_k 和 threshold 有默认值"""
    print(f"查询: {query}, 返回 {top_k} 条, 阈值: {threshold}")

search("什么是RAG")                # 用默认值：top_k=3, threshold=0.5
search("什么是RAG", top_k=5)       # 只改 top_k
search("什么是RAG", 5, 0.8)        # 全部指定（按位置）
```

### 2.3 关键字参数

调用时指定参数名，不用管顺序：

```python
def create_prompt(question, context, system_msg="你是一个知识库助手"):
    return f"系统: {system_msg}\n上下文: {context}\n问题: {question}"

# 关键字参数，顺序随意
prompt = create_prompt(
    context="RAG是检索增强生成技术",
    question="什么是RAG？"
)
```

### 2.4 可变参数 `*args` 和 `**kwargs`

```python
# *args — 接收任意数量的位置参数，打包成元组
def log_messages(*messages):
    for msg in messages:
        print(f"[LOG] {msg}")

log_messages("开始处理", "切分文档", "生成向量")

# **kwargs — 接收任意数量的关键字参数，打包成字典
def build_metadata(**kwargs):
    return kwargs

meta = build_metadata(source="doc.pdf", page=5, chunk_id="c001")
# {"source": "doc.pdf", "page": 5, "chunk_id": "c001"}
```

## 3. 返回值：函数的"输出"

### 3.1 基本返回

```python
def calculate_similarity(vec1, vec2):
    """计算两个向量的余弦相似度（简化版）"""
    # 这里只是示意，实际会用 numpy
    similarity = 0.85
    return similarity        # 用 return 把结果送出去

score = calculate_similarity([1, 2, 3], [1, 2, 4])
```

### 3.2 返回多个值

Python 函数可以返回多个值（本质是返回一个元组）：

```python
def load_and_split(file_path):
    """加载文档并切分，返回原文档和切分结果"""
    content = "这是文档内容..."
    chunks = ["这是", "文档", "内容..."]
    return content, chunks     # 返回两个值

doc_content, doc_chunks = load_and_split("test.txt")    # 解包
```

### 3.3 提前返回

```python
def validate_query(query):
    """验证用户查询是否合法"""
    if not query:
        return False, "查询不能为空"
    if len(query) > 1000:
        return False, "查询太长，请缩短"
    return True, "验证通过"

is_valid, message = validate_query("")
if not is_valid:
    print(f"错误: {message}")    # 错误: 查询不能为空
```

## 4. 变量作用域

```python
# 全局变量 — 在函数外面定义
API_KEY = "sk-xxx123"

def call_api():
    # 函数内部可以读取全局变量
    print(f"使用 API Key: {API_KEY}")

    # 但如果要修改全局变量，需要加 global 声明
    # （不推荐，尽量避免修改全局变量）

def process():
    # 局部变量 — 只在函数内部有效
    local_result = "处理完成"
    print(local_result)

# print(local_result)  # ❌ 报错！函数外面访问不到局部变量
```

**面试相关：** RAG 项目中，API Key、模型名称等配置通常作为全局常量（大写命名），而函数内部的中间结果用局部变量。

## 5. 函数作为参数（高阶函数）

Python 中函数是"一等公民"——可以像变量一样传来传去。这在 LangChain 中大量使用。

```python
def apply_to_chunks(chunks, transform_func):
    """对每个文本块应用一个变换函数"""
    return [transform_func(chunk) for chunk in chunks]

# 定义不同的变换函数
def to_lowercase(text):
    return text.lower()

def add_prefix(text):
    return f"[文档片段] {text}"

chunks = ["Hello World", "RAG is Great"]

# 把函数当作参数传进去
result1 = apply_to_chunks(chunks, to_lowercase)
# ["hello world", "rag is great"]

result2 = apply_to_chunks(chunks, add_prefix)
# ["[文档片段] Hello World", "[文档片段] RAG is Great"]
```

## 6. Lambda 匿名函数

简单的一行函数，不用 `def` 起名：

```python
# 普通函数
def double(x):
    return x * 2

# 等价的 lambda
double = lambda x: x * 2

# 常见用途：排序时指定排序键
results = [
    {"content": "文档A", "score": 0.7},
    {"content": "文档B", "score": 0.95},
    {"content": "文档C", "score": 0.85},
]

# 按 score 从高到低排序
sorted_results = sorted(results, key=lambda r: r["score"], reverse=True)
# [{"content": "文档B", "score": 0.95}, {"content": "文档C", "score": 0.85}, ...]
```

## 7. 在 RAG 项目中的应用场景

| 函数概念 | RAG 场景中的具体用途 |
|---|---|
| 基本函数 | `load_document(path)` — 加载文档、`split_text(text)` — 切分文本 |
| 默认参数 | `search(query, top_k=5, threshold=0.5)` — 可配置的检索参数 |
| 返回多个值 | `process_query()` 同时返回处理后的问题和过滤掉的停用词 |
| 函数作参数 | LangChain 的管道 `\|` 操作符本质就是把函数串联起来 |
| Lambda | 排序检索结果、列表推导式中的简单变换 |
| 可变参数 | `build_prompt(*context_chunks)` — 动态拼接不定数量的上下文 |

## 8. 代码组织：从脚本到函数

初学者常见的"面条代码"：

```python
# ❌ 所有逻辑堆在一起
text = open("doc.txt").read()
chunks = [text[i:i+500] for i in range(0, len(text), 500)]
for chunk in chunks:
    print(chunk)
```

改成函数化：

```python
# ✅ 每个函数做一件事，组合起来就是 RAG 管道
def load_document(file_path):
    """加载文档"""
    with open(file_path, "r", encoding="utf-8") as f:
        return f.read()

def split_text(text, chunk_size=500):
    """切分文本"""
    return [text[i:i+chunk_size] for i in range(0, len(text), chunk_size)]

def process_document(file_path):
    """完整的文档处理流程"""
    text = load_document(file_path)
    chunks = split_text(text)
    return chunks

# 调用
chunks = process_document("doc.txt")
for chunk in chunks:
    print(chunk)
```

**这就是函数的核心价值：把复杂问题拆成小块，每个小块独立可测试、可复用。**
