# 条件判断与循环

> 条件判断让程序能"做选择"，循环让程序能"重复干活"。RAG 系统中：根据相似度分数决定是否返回结果（条件判断），遍历所有文档块进行切分（循环）。

---

## 1. 条件判断：让程序会"做选择"

### 1.1 基本 if/elif/else

```python
similarity_score = 0.85

if similarity_score > 0.8:
    print("高度相关，直接返回")
elif similarity_score > 0.5:
    print("有一定相关性，需要重排")
else:
    print("相关性太低，不返回给用户")
```

**执行逻辑：** 从上往下依次检查条件，一旦某个条件为 `True`，执行对应的代码块，**跳过剩余所有分支**。

### 1.2 多条件组合

```python
score = 0.85
source = "manual.pdf"

# and：两个条件都满足
if score > 0.8 and source == "manual.pdf":
    print("来自手册且高度相关")

# or：至少一个条件满足
if score > 0.9 or source == "faq.txt":
    print("要么特别相关，要么来自FAQ")

# not：取反
if not (score < 0.5):
    print("相关性还行（不低于0.5）")
```

### 1.3 条件表达式（三元运算符）

简洁版的 if/else，适合简单的二选一：

```python
score = 0.85
label = "相关" if score > 0.5 else "不相关"    # 一行搞定
```

### 1.4 实战：处理 API 返回结果

```python
response = {"status": "success", "answer": "RAG是检索增强生成"}

if response.get("status") == "success":
    print(f"回答：{response['answer']}")
elif response.get("status") == "error":
    print(f"出错了：{response.get('message', '未知错误')}")
else:
    print("未知状态")
```

## 2. 循环：让程序会"重复干活"

### 2.1 for 循环 — 遍历集合

**这是 RAG 项目中最常用的循环**，用于遍历文档列表、检索结果、对话历史等。

```python
# 遍历列表
docs = ["文档A", "文档B", "文档C"]
for doc in docs:
    print(f"正在处理：{doc}")

# 遍历字典
metadata = {"source": "manual.pdf", "page": 5, "chunk_id": "c001"}
for key, value in metadata.items():
    print(f"{key} = {value}")

# range() — 按次数循环
for i in range(5):           # 0, 1, 2, 3, 4
    print(f"第 {i} 次循环")

# enumerate() — 同时拿到索引和值（遍历时要知道"第几个"）
chunks = ["块1", "块2", "块3"]
for index, chunk in enumerate(chunks):
    print(f"第 {index} 个块：{chunk}")
```

### 2.2 while 循环 — 条件满足就一直做

```python
# 重试机制示例：调用 API 失败时重试
retry_count = 0
max_retries = 3
success = False

while retry_count < max_retries and not success:
    print(f"第 {retry_count + 1} 次尝试...")
    # 这里调用 API（模拟）
    # success = call_api()
    retry_count += 1

if not success:
    print("重试次数用完，放弃")
```

### 2.3 循环控制：break、continue

```python
# break — 立即跳出循环
scores = [0.3, 0.4, 0.85, 0.9, 0.7]
for score in scores:
    if score > 0.8:
        print(f"找到高分结果：{score}")
        break                # 找到第一个就停，不再继续

# continue — 跳过当前这一轮，继续下一轮
texts = ["你好", "", "世界", "", "RAG"]
for text in texts:
    if text == "":           # 跳过空字符串
        continue
    print(f"处理：{text}")
```

### 2.4 列表推导式 — for 循环的"语法糖"

```python
# 传统写法
scores = [0.9, 0.3, 0.8, 0.4, 0.95]
high_scores = []
for s in scores:
    if s > 0.7:
        high_scores.append(s)
# high_scores = [0.9, 0.8, 0.95]

# 列表推导式（一行搞定）
high_scores = [s for s in scores if s > 0.7]
# [0.9, 0.8, 0.95]

# 实战：把文档列表中的文本全部提取出来
docs = [
    {"content": "第一段", "score": 0.9},
    {"content": "第二段", "score": 0.3},
    {"content": "第三段", "score": 0.8},
]
relevant_texts = [doc["content"] for doc in docs if doc["score"] > 0.5]
# ["第一段", "第三段"]
```

## 3. 嵌套使用：条件 + 循环的组合

这是 RAG 系统中的典型模式——遍历检索结果，根据条件筛选：

```python
search_results = [
    {"content": "RAG是检索增强生成", "score": 0.92, "source": "doc1.pdf"},
    {"content": "向量数据库存储Embedding", "score": 0.45, "source": "doc2.pdf"},
    {"content": "LangChain是大模型框架", "score": 0.88, "source": "doc1.pdf"},
    {"content": "Docker容器化部署", "score": 0.31, "source": "doc3.pdf"},
]

# 筛选出高分结果，并格式化
relevant_chunks = []
for result in search_results:
    if result["score"] > 0.7:
        formatted = f"[来源: {result['source']}] {result['content']}"
        relevant_chunks.append(formatted)

# 输出筛选结果
for chunk in relevant_chunks:
    print(chunk)
# [来源: doc1.pdf] RAG是检索增强生成
# [来源: doc1.pdf] LangChain是大模型框架
```

## 4. 在 RAG 项目中的应用场景

| 语法结构 | RAG 场景中的具体用途 |
|---|---|
| `if/elif/else` | 根据相似度分数筛选检索结果、判断 API 返回状态、根据用户意图路由 |
| `for` 遍历列表 | 遍历文档进行切分、遍历检索结果格式化、遍历对话历史 |
| `for` 遍历字典 | 处理 API 响应 JSON、读取文档元数据 |
| `while` | API 调用失败重试、流式输出循环读取 |
| `break` | 找到第一个满足条件的结果就停止 |
| `continue` | 跳过空文档、跳过已处理的文件 |
| 列表推导式 | 批量提取字段、批量过滤、批量转换格式 |
