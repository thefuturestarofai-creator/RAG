# 变量与数据类型

> 这是 Python 的"地基中的地基"。RAG 项目中，你处理的每一段文本、每一个检索结果、每一条 API 响应，都是变量和数据类型的组合。

---

## 1. 变量：给数据起名字

变量就是一个**名字标签**，贴在数据上面，方便你以后找到它。

```python
name = "RAG"          # 用 name 这个标签记住 "RAG" 这串文字
score = 95.5          # 用 score 这个标签记住 95.5 这个数字
is_ready = True       # 用 is_ready 这个标签记住 True 这个状态
```

**命名规则：**
- 只能包含字母、数字、下划线 `_`
- 不能以数字开头（`1name` ❌，`name1` ✅）
- 区分大小写（`Name` 和 `name` 是两个不同的变量）
- 推荐用**小写+下划线**风格：`my_variable`（这是 Python 社区的惯例，叫 snake_case）

---

## 2. 六大核心数据类型

### 2.1 字符串 `str` — 文本数据

RAG 项目中最常见的类型：用户的问题、文档的内容、LLM 的回答，全是字符串。

```python
question = "什么是RAG？"
answer = '检索增强生成是一种技术'    # 单引号双引号都行
doc = """这是
一段多行
文本"""                              # 三引号可以写多行
```

**常用操作：**

```python
# 拼接
greeting = "你好，" + "世界"         # "你好，世界"

# 格式化（f-string）—— 最常用，RAG 项目里构造 Prompt 时天天用
name = "GPT"
prompt = f"请用一句话解释：{name}"    # "请用一句话解释：GPT"

# 常用方法
text = "  Hello, RAG World!  "
text.strip()           # 去掉首尾空格 → "Hello, RAG World!"
text.lower()           # 全部小写 → "  hello, rag world!  "
text.split(",")        # 按逗号分割 → ['  Hello', ' RAG World!  ']
len(text)              # 长度 → 20
"RAG" in text          # 是否包含 → True
```

### 2.2 整数 `int` 和浮点数 `float` — 数字数据

```python
chunk_size = 500            # 整数：Chunk 的大小（Token 数）
similarity = 0.87           # 浮点数：相似度分数
temperature = 0.1           # 浮点数：模型的"创意度"

# 基本运算
a = 10
b = 3
a + b     # 加 → 13
a - b     # 减 → 7
a * b     # 乘 → 30
a / b     # 除 → 3.333...
a // b    # 整除 → 3
a % b     # 取余 → 1
a ** b    # 幂运算 → 1000
```

### 2.3 布尔 `bool` — 真或假

只有两个值：`True` 和 `False`。在条件判断中大量使用。

```python
is_relevant = True       # 检索结果是否相关
has_error = False        # API 调用是否出错

# 比较运算产生布尔值
score = 0.85
score > 0.8              # True
score == 0.9             # False（== 是判断相等，= 是赋值）
score != 0.5             # True（!= 是"不等于"）

# 逻辑运算
True and False           # False（两个都为 True 才是 True）
True or False            # True（有一个 True 就是 True）
not True                 # False（取反）
```

### 2.4 列表 `list` — 有序的数据集合

RAG 项目中**最核心的数据结构之一**：检索到的多个文档块、对话历史、切分后的文本片段，都是列表。

```python
# 创建列表
chunks = ["第一段文本", "第二段文本", "第三段文本"]
scores = [0.95, 0.87, 0.72]
mixed = ["text", 42, True, 3.14]        # 可以混着放（但不推荐）

# 访问元素（从 0 开始数！）
chunks[0]            # "第一段文本"（第一个）
chunks[2]            # "第三段文本"（第三个）
chunks[-1]           # "第三段文本"（倒数第一个）

# 切片（取一部分）
chunks[0:2]          # ["第一段文本", "第二段文本"]（取第 0、1 个，不包含第 2 个）

# 常用方法
chunks.append("第四段文本")    # 在末尾添加
chunks.pop()                   # 取出并删除最后一个
len(chunks)                    # 长度
"第一段文本" in chunks         # True（是否包含）

# 遍历（RAG 项目中极其常用：遍历检索结果、遍历文档块）
for 变量 in 列表:
    # 对每个元素做操作
    print(变量)
执行流程： 从列表中依次取出每个元素，赋值给变量，然后执行缩进的代码块。
for chunk in chunks:
    print(chunk)

# 列表推导式（Python 的"语法糖"，简洁高效）
[表达式 for 变量 in 列表] = 遍历列表，对每个元素执行表达式，结果收集到新列表
texts = ["Hello", "World", "RAG"]
lengths = [len(t) for t in texts]    # [5, 5, 3]
```

### 2.5 字典 `dict` — 键值对数据

RAG 项目中**另一个核心数据结构**：API 请求/响应的 JSON 数据、文档的元数据（metadata），都是字典。

```python
# 创建字典
doc = {
    "content": "RAG是检索增强生成技术",
    "source": "knowledge_base.pdf",
    "page": 3,
    "score": 0.92
}

# 访问值
doc["content"]         # "RAG是检索增强生成技术"
doc.get("score", 0)    # 0.92（如果 key 不存在，返回默认值 0，不会报错）

# 修改和添加
doc["score"] = 0.95              # 修改
doc["chunk_id"] = "chunk_001"    # 添加新键值对

# 遍历
for key, value in doc.items():
    print(f"{key}: {value}")
```

### 2.6 空值 `None` — 表示"没有"

```python
result = None    # API 还没返回结果时，先用 None 占位

# 判断是否为 None
if result is None:
    print("还没获取到结果")
```

## 3. 类型转换

不同类型之间可以互相转换，这在处理 API 数据时非常常见：

```python
# 字符串 ↔ 数字
num_str = "42"
num = int(num_str)           # 字符串 → 整数
back = str(num)              # 整数 → 字符串

# 字符串 ↔ 列表
text = "hello,world,rag"
parts = text.split(",")      # 字符串 → 列表: ["hello", "world", "rag"]
joined = ",".join(parts)     # 列表 → 字符串: "hello,world,rag"

# 查看类型
type("hello")     # <class 'str'>
type(42)          # <class 'int'>
type(True)        # <class 'bool'>
type([1, 2])      # <class 'list'>
```

## 4. 在 RAG 项目中的应用场景

| 数据类型 | RAG 场景中的具体用途 |
|---|---|
| `str` | 用户提问、文档内容、LLM 回答、Prompt 模板 |
| `int` | Chunk Size（块大小）、Top-K（返回数量）、Token 计数 |
| `float` | 相似度分数、Temperature、权重系数 |
| `bool` | 标记检索是否成功、是否需要重试、文档是否已处理 |
| `list` | 检索结果列表、对话历史、切分后的文本块集合 |
| `dict` | API 请求/响应 JSON、文档元数据 metadata |
| `None` | 初始化未赋值的变量、表示"没有找到结果" |
