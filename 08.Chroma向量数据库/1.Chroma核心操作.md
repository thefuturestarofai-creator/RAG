# Chroma 核心操作

## 什么是 Chroma？

想象一个**智能图书馆**：你把书（文档）放进去时，图书馆会自动理解每本书的内容，按照内容的相似性来组织。当你问一个问题时，图书馆能快速找到与你问题最相关的几本书。

Chroma 就是这样一个专门为 AI 应用设计的向量数据库，它的特点是：
- 简单易用（几行代码就能上手）
- 本地运行（无需外部服务）
- 支持持久化（数据可以保存到磁盘）
- 与 LangChain 无缝集成

## 核心概念

### 1. Collection（集合）

Collection 是 Chroma 中存储数据的基本单位，类似于传统数据库中的"表"：

```python
import chromadb

client = chromadb.Client()  # 创建客户端

# 创建集合
collection = client.create_collection(
    name="my_documents",
    metadata={"description": "我的文档集合"}
)
```

### 2. 添加文档（add）

向集合中添加文档：

```python
collection.add(
    documents=["文档1的内容", "文档2的内容"],           # 文档内容
    metadatas=[{"source": "file1.txt"}, {"source": "file2.txt"}],  # 元数据
    ids=["doc1", "doc2"]                                 # 唯一标识
)
```

### 3. 查询文档（query）

根据问题检索相关文档：

```python
results = collection.query(
    query_texts=["用户的问题"],    # 查询文本
    n_results=3                     # 返回前3个结果
)

# 返回结果包含：
# - documents: 文档内容
# - distances: 相似度距离
# - metadatas: 元数据
# - ids: 文档ID
```

### 4. 删除文档（delete）

```python
# 按 ID 删除
collection.delete(ids=["doc1"])

# 按条件删除
collection.delete(where={"source": "file1.txt"})
```

### 5. Embedding Function

Chroma 需要将文本转换为向量，这个过程通过 Embedding Function 实现：

```python
# 使用 Chroma 内置的默认 Embedding（不需要 API Key）
collection = client.create_collection("my_collection")

# 使用自定义 Embedding（如 OpenAI）
from chromadb.utils.embedding_functions import OpenAIEmbeddingFunction

ef = OpenAIEmbeddingFunction(
    api_key="your-api-key",
    model_name="text-embedding-3-small"
)
collection = client.create_collection("my_collection", embedding_function=ef)
```

### 6. 元数据过滤

可以在查询时按元数据过滤：

```python
# 只在特定来源中检索
results = collection.query(
    query_texts=["问题"],
    n_results=3,
    where={"source": "file1.txt"}  # 过滤条件
)

# 复杂过滤
results = collection.query(
    query_texts=["问题"],
    where={
        "$and": [
            {"source": {"$eq": "file1.txt"}},
            {"page": {"$gte": 10}}
        ]
    }
)
```

### 7. 持久化存储

默认情况下，Chroma 的数据只在内存中。要持久化到磁盘：

```python
# 方式1：使用 PersistentClient
client = chromadb.PersistentClient(path="./chroma_db")

# 方式2：使用 HttpClient（连接远程 Chroma 服务）
client = chromadb.HttpClient(host="localhost", port=8000)
```

## 在 RAG 项目中的应用场景

1. **知识库存储**：将切分后的文档向量化并存储到 Chroma
2. **语义检索**：用户提问时，从 Chroma 中检索最相关的文档
3. **文档管理**：通过元数据管理文档的来源、更新时间等信息
4. **增量更新**：当知识库更新时，可以添加、修改或删除特定文档

## 面试常问问题

### Q1: Chroma 和传统数据库（如 MySQL）有什么区别？

**答**：主要区别：1）数据类型不同——MySQL 存储结构化数据，Chroma 存储向量数据。2）查询方式不同——MySQL 用精确匹配（WHERE），Chroma 用相似度检索。3）用途不同——MySQL 适合精确查询，Chroma 适合语义搜索。4）索引方式不同——MySQL 用 B-Tree 索引，Chroma 用 HNSW 等向量索引。

### Q2: Chroma 的元数据有什么作用？

**答**：元数据是附加在文档上的键值对信息，作用：1）过滤检索——可以在查询时按元数据过滤，缩小检索范围。2）来源追踪——记录文档的来源文件、页码等信息。3）分类管理——按类别、标签等组织文档。4）业务逻辑——存储更新时间、版本号等业务信息。

### Q3: Chroma 如何选择 Embedding Function？

**答**：Chroma 支持多种 Embedding Function：1）默认使用内置的 all-MiniLM-L6-v2 模型，无需 API Key，适合英文场景。2）中文场景推荐使用 OpenAI 的 text-embedding-ada-002。3）也可以使用 HuggingFace 上的开源模型。选择依据：语言支持、精度要求、成本预算。
