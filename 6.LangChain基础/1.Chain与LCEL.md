# Chain 与 LCEL

## 什么是 LangChain？

LangChain 是一个用于构建大语言模型应用的框架。它提供了一套工具和抽象，让开发者可以更方便地将 LLM 与其他组件（如数据库、API、文件等）组合在一起。

类比：如果 LLM 是一块块乐高积木，LangChain 就是教你如何把这些积木拼成城堡的说明书。

## 核心概念

### 1. Model（模型）

LangChain 对各种 LLM 进行了统一封装：

```python
from langchain_openai import ChatOpenAI

# 创建模型实例
llm = ChatOpenAI(
    api_key="your-api-key",
    base_url="https://api.openai.com/v1",
    model="gpt-4o-mini",
    temperature=0.7
)
```

### 2. PromptTemplate（提示模板）

把提示词"模板化"，方便复用：

```python
from langchain_core.prompts import PromptTemplate

# 创建模板
template = PromptTemplate.from_template(
    "请用{style}的风格解释{topic}"
)

# 填充变量
prompt = template.invoke({
    "style": "通俗易懂",
    "topic": "什么是机器学习"
})
```

类比：PromptTemplate 就像一个"填空题模板"，你只需要填写空白处的内容。

### 3. Chain（链）

Chain 是将多个组件串联起来的工作流：

```python
# 创建一个简单的 Chain：PromptTemplate → LLM
chain = prompt_template | llm

# 运行 Chain
result = chain.invoke({"topic": "机器学习"})
```

### 4. LCEL 管道语法（LangChain Expression Language）

LCEL 使用 `|` 管道符将组件连接，就像 Unix 的管道命令：

```python
from langchain_core.prompts import PromptTemplate
from langchain_openai import ChatOpenAI

# 用 | 将组件连接成管道
chain = (
    PromptTemplate.from_template("请解释{topic}")  # 第一步：格式化提示
    | ChatOpenAI(model="gpt-4o-mini")            # 第二步：调用模型
)

# 运行管道
result = chain.invoke({"topic": "向量数据库"})
```

类比：LCEL 管道就像工厂的流水线，原材料（输入）经过一道道工序（组件），最终变成成品（输出）。

## LCEL 的优势

1. **流式输出**：`chain.stream()` 支持流式返回
2. **批量处理**：`chain.batch()` 可以批量处理多个输入
3. **异步支持**：`chain.ainvoke()` 支持异步调用
4. **可组合性**：可以轻松添加或移除管道中的步骤

## 在 RAG 项目中的应用场景

1. **查询处理链**：用户问题 → 查询改写 → Embedding → 检索 → 生成答案
2. **文档处理链**：文档加载 → 切分 → 向量化 → 存储
3. **问答链**：将 PromptTemplate + LLM + OutputParser 组合成问答管道
4. **多步推理链**：对于复杂问题，可以用 Chain 实现多步推理

## 面试常问问题

### Q1: LCEL 是什么？相比直接调用 API 有什么优势？

**答**：LCEL（LangChain Expression Language）是 LangChain 的管道语法，使用 `|` 管道符将组件连接。相比直接调用 API，LCEL 的优势：1）声明式定义，代码更清晰；2）内置流式输出、批量处理、异步支持；3）易于组合和复用；4）可以无缝添加日志、缓存、重试等横切关注点。

### Q2: Chain 和直接写函数调用有什么区别？

**答**：Chain 是 LangChain 的抽象概念，它把多个步骤封装成一个可复用的管道。相比直接写函数：1）Chain 支持标准接口（invoke/stream/batch）；2）Chain 可以嵌套组合；3）Chain 可以利用 LangChain 生态的各种工具（如回调、监控）；4）Chain 的定义更声明式，更容易理解整体流程。

### Q3: PromptTemplate 的作用是什么？

**答**：PromptTemplate 将提示词模板化，支持变量替换。好处：1）复用性——同一个模板可以用于不同输入；2）可维护性——修改模板一处，所有调用都生效；3）标准化——团队可以共享和审查提示模板；4）与 LCEL 管道无缝集成。

### Q4: LangChain 和直接调用 OpenAI API 相比，优势和劣势是什么？

**答**：**优势**：1）统一封装——LangChain 对接了上百种 LLM，切换模型只需改一行代码；2）组件化——Prompt、LLM、OutputParser 等可以自由组合；3）生态丰富——内置文档加载、向量存储、检索等 RAG 全套工具；4）LCEL 管道——声明式定义工作流，支持流式、批量、异步。**劣势**：1）抽象层开销——多一层封装，调试时链路更长；2）版本不稳定——LangChain 更新频繁，API 经常变动；3）学习成本——概念多（Chain、Agent、Memory 等），入门门槛高；4）性能损耗——简单场景下，直接调用 API 更轻量、更快。**选择建议**：简单场景直接调用 API，复杂 RAG 项目用 LangChain。
