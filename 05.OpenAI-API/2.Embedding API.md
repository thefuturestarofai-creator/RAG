# Embedding API

## 什么是 Embedding？

想象每个文本都有一个**语义 GPS 坐标**。就像地球上每个地点都有经纬度坐标一样，每段文字通过 Embedding 会被转换成一组数字（向量），这组数字就代表了这段文字的"语义位置"。

- 意思相近的文字，它们的"坐标"会很接近
- 意思不同的文字，它们的"坐标"会相距很远

## 核心概念

### 1. 什么是向量（Vector）？

向量就是一组有序的数字，例如：`[0.1, -0.3, 0.5, 0.8, ...]`

在 Embedding 中：
- 一段文本 → 一个向量（通常有 1536 个数字，即 1536 维）
- 这个向量代表了文本的"语义信息"

类比：就像把一篇文章压缩成一个"指纹"，这个指纹能反映文章的核心含义。

### 2. client.embeddings.create() 使用方法

```python
from openai import OpenAI

client = OpenAI(api_key="your-api-key")

response = client.embeddings.create(
    model="text-embedding-3-small",  # 推荐模型：text-embedding-3-small（便宜、效果好）
    input="你好，世界"                 # 要转换的文本
)

# 获取向量
vector = response.data[0].embedding  # 一个包含 1536 个数字的列表
```

> **模型选择建议**：Embedding 场景推荐 `text-embedding-3-small`（性价比最高，1536维）。需要更高精度时可用 `text-embedding-3-large`（3072维）。不建议再使用旧版 text-embedding-ada-002。

### 3. 余弦相似度（Cosine Similarity）

余弦相似度用来衡量两个向量的"方向相似性"：

- **值为 1**：两个向量方向完全相同（语义完全一致）
- **值为 0**：两个向量垂直（语义无关）
- **值为 -1**：两个向量方向完全相反（语义相反）

计算公式：

```
cosine_similarity = (A · B) / (|A| × |B|)
```

用 numpy 实现：

```python
import numpy as np

def cosine_similarity(vec_a, vec_b):
    """计算两个向量的余弦相似度"""
    a = np.array(vec_a)
    b = np.array(vec_b)
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
```

类比：余弦相似度就像比较两个箭头的指向，指向越接近，相似度越高。

### 4. Embedding 的维度

不同 Embedding 模型产生不同维度的向量：

| 模型 | 维度 | 特点 |
|------|------|------|
| text-embedding-ada-002 | 1536 | OpenAI 经典模型 |
| text-embedding-3-small | 1536 | 更新、更便宜 |
| text-embedding-3-large | 3072 | 更高精度 |

## 在 RAG 项目中的应用场景

1. **文档向量化**：将知识库中的文档切分成小块后，用 Embedding API 转换成向量存储到向量数据库中
2. **查询向量化**：用户提问时，将问题也转换成向量
3. **语义检索**：通过计算用户问题向量与文档向量的余弦相似度，找到最相关的文档片段
4. **去重与聚类**：利用向量相似度，去除重复文档或将相似文档归类

这是 RAG 系统中最关键的技术之一——没有 Embedding，就无法实现"语义检索"。

## 面试常问问题

### Q1: Embedding 和 One-Hot 编码有什么区别？

**答**：One-Hot 编码是稀疏的、高维的，每个词都是独立的，无法表达语义关系（"猫"和"狗"的距离与"猫"和"汽车"的距离相同）。Embedding 是稠密的、低维的，能捕捉语义信息——意思相近的词在向量空间中距离更近。Embedding 是通过大规模语料训练出来的，蕴含了丰富的语言知识。

### Q2: 为什么 RAG 需要 Embedding？直接关键词搜索不行吗？

**答**：关键词搜索只能匹配字面相同的词，无法理解语义。例如搜索"怎么减肥"，关键词搜索找不到包含"如何瘦身"的文档，但 Embedding 可以理解它们是同一个意思。Embedding 实现的是"语义检索"，能理解同义词、近义词、不同表达方式，大大提高了检索的准确性和召回率。

### Q3: 余弦相似度和欧氏距离有什么区别？为什么 RAG 通常用余弦相似度？

**答**：欧氏距离衡量两点之间的绝对距离，受向量长度影响；余弦相似度衡量两个向量的方向相似性，不受长度影响。RAG 中用余弦相似度是因为：1）文档长度不同会导致向量"长度"不同，但语义方向是一致的；2）余弦相似度的值在 [-1, 1] 之间，便于理解和设置阈值；3）计算效率高，适合大规模检索。
