# 文本预处理

## 一句话理解

文本预处理就像"洗菜"——从菜市场买回来的菜（原始文档）有泥沙、烂叶子、农药残留（特殊字符、乱码、多余空白），你得先洗干净、切好，才能下锅炒（切分和向量化）。跳过这一步，后面的"菜"味道会很差。

---

## 一、为什么需要预处理？

### 原始文档的常见问题

| 问题 | 示例 | 影响 |
|------|------|------|
| 特殊字符 | `\x00`、`﻿`（BOM） | 切分器可能出错，向量化结果异常 |
| 多余空白 | 连续空格、空行 | 占用 token，浪费切分空间 |
| 编码问题 | GBK/UTF-8 混用 | 中文乱码 |
| 噪声内容 | 页眉页脚、版权声明、广告 | 检索时被误匹配，降低回答质量 |
| 格式不统一 | "2024年" vs "2024 年" | 相同内容被当成不同文本 |

### 预处理的价值

```
预处理前：检索准确率 60%
    ↓ 去除噪声、标准化格式
预处理后：检索准确率 85%
```

## 二、清洗（Cleaning）

### 1. 去除特殊字符

```python
import re

def remove_special_chars(text: str) -> str:
    """去除控制字符，保留常用标点和换行"""
    # 去除 ASCII 控制字符（\x00-\x1f 中除了 \n \r \t）
    text = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f]', '', text)
    # 去除 BOM（Byte Order Mark）
    text = text.replace('﻿', '')
    # 去除零宽字符
    text = re.sub(r'[​‌‍﻿]', '', text)
    return text
```

**类比**：特殊字符就像菜里的小石子，咬到会崩牙（程序报错），必须挑出来。

### 2. 处理多余空白

```python
def normalize_whitespace(text: str) -> str:
    """标准化空白字符"""
    # 将制表符替换为空格
    text = text.replace('\t', ' ')
    # 多个连续空格 → 一个空格
    text = re.sub(r' +', ' ', text)
    # 多个连续换行 → 两个换行（保留段落分隔）
    text = re.sub(r'\n{3,}', '\n\n', text)
    # 去除每行首尾空白
    lines = [line.strip() for line in text.split('\n')]
    return '\n'.join(lines)
```

**类比**：多余空白就像菜里的水分——洗完要沥干，不然炒菜会溅油（浪费 token 空间）。

### 3. 去除噪声内容

```python
def remove_noise(text: str) -> str:
    """去除常见的噪声内容"""
    # 去除页眉页脚格式（如 "第 X 页"、"Page X"）
    text = re.sub(r'第\s*\d+\s*页', '', text)
    text = re.sub(r'Page\s*\d+', '', text, flags=re.IGNORECASE)

    # 去除版权声明
    text = re.sub(r'版权所有.*?保留所有权利', '', text)
    text = re.sub(r'Copyright.*?All rights reserved', '', text, flags=re.IGNORECASE)

    # 去除 URL（如果不需要）
    # text = re.sub(r'https?://\S+', '', text)

    return text.strip()
```

## 三、标准化（Normalization）

### 1. 全角半角统一

中文文档中经常混用全角和半角字符：

```python
def normalize_width(text: str) -> str:
    """全角字符转半角"""
    result = []
    for char in text:
        code = ord(char)
        # 全角空格特殊处理
        if code == 0x3000:
            result.append(' ')
        # 其他全角字符（！～ 之间）
        elif 0xFF01 <= code <= 0xFF5E:
            result.append(chr(code - 0xFEE0))
        else:
            result.append(char)
    return ''.join(result)

# 示例
print(normalize_width("Ｈｅｌｌｏ，Ｗｏｒｌｄ！"))  # → "Hello,World!"
print(normalize_width("２０２４年"))  # → "2024年"
```

### 2. 中文标点统一

```python
def normalize_punctuation(text: str) -> str:
    """统一中文标点为英文标点（或反之，看需求）"""
    mapping = {
        '，': ', ',
        '。': '. ',
        '！': '! ',
        '？': '? ',
        '：': ': ',
        '；': '; ',
        '（': '(',
        '）': ') ',
    }
    for cn, en in mapping.items():
        text = text.replace(cn, en)
    return text
```

**注意**：是否统一标点取决于你的需求。如果文档主要是中文，保留中文标点可能更好。

### 3. 数字和日期格式统一

```python
def normalize_numbers(text: str) -> str:
    """统一数字格式"""
    # 中文数字 → 阿拉伯数字（简单示例）
    cn_num_map = {'一': '1', '二': '2', '三': '3', '四': '4', '五': '5'}
    for cn, ar in cn_num_map.items():
        text = text.replace(cn, ar)
    return text
```

## 四、编码处理

### 常见编码问题

```python
def safe_read_file(file_path: str) -> str:
    """安全读取文件，自动处理编码"""
    encodings = ['utf-8', 'gbk', 'gb2312', 'utf-16', 'latin-1']

    for encoding in encodings:
        try:
            with open(file_path, 'r', encoding=encoding) as f:
                return f.read()
        except (UnicodeDecodeError, UnicodeError):
            continue

    # 所有编码都失败，用 errors='ignore' 强制读取
    with open(file_path, 'r', encoding='utf-8', errors='ignore') as f:
        return f.read()
```

**类比**：编码就像语言——有些文件是用"英语"（UTF-8）写的，有些是用"方言"（GBK）写的，你需要尝试不同的"翻译器"才能读懂。

## 五、完整的预处理管道

```python
from typing import List
from langchain.schema import Document

class TextPreprocessor:
    """文本预处理管道：清洗 → 标准化"""

    def __init__(self, remove_noise: bool = True):
        self.remove_noise = remove_noise

    def process(self, documents: List[Document]) -> List[Document]:
        """处理文档列表"""
        processed = []
        for doc in documents:
            text = doc.page_content

            # 1. 清洗
            text = self._remove_special_chars(text)
            text = self._normalize_whitespace(text)

            if self.remove_noise:
                text = self._remove_noise(text)

            # 2. 标准化
            text = self._normalize_width(text)

            # 3. 跳过空文档
            if not text.strip():
                continue

            processed.append(Document(
                page_content=text,
                metadata=doc.metadata.copy()
            ))

        return processed

    def _remove_special_chars(self, text: str) -> str:
        import re
        text = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f]', '', text)
        text = text.replace('﻿', '')
        return text

    def _normalize_whitespace(self, text: str) -> str:
        import re
        text = text.replace('\t', ' ')
        text = re.sub(r' +', ' ', text)
        text = re.sub(r'\n{3,}', '\n\n', text)
        lines = [line.strip() for line in text.split('\n')]
        return '\n'.join(lines)

    def _remove_noise(self, text: str) -> str:
        import re
        text = re.sub(r'第\s*\d+\s*页', '', text)
        text = re.sub(r'Page\s*\d+', '', text, flags=re.IGNORECASE)
        return text.strip()

    def _normalize_width(self, text: str) -> str:
        result = []
        for char in text:
            code = ord(char)
            if code == 0x3000:
                result.append(' ')
            elif 0xFF01 <= code <= 0xFF5E:
                result.append(chr(code - 0xFEE0))
            else:
                result.append(char)
        return ''.join(result)
```

## 六、预处理的注意事项

### 1. 不要过度清洗

```
❌ 错误：把所有标点都去掉 → "2024年营收增长20" 变成 "2024年营收增长20"
❌ 错误：把所有数字都去掉 → 丢失了重要信息
✅ 正确：只去除明确的噪声，保留有意义的内容
```

### 2. 保留语义完整性

```
❌ 错误：每行单独处理，把跨行的句子切断
✅ 正确：先判断段落边界，再做行级处理
```

### 3. 记录处理日志

```python
logger.info(f"原始文档: {len(text)} 字符")
logger.info(f"清洗后: {len(cleaned_text)} 字符，去除 {len(text) - len(cleaned_text)} 字符")
```

## 七、面试常问问题

### Q1: 文本预处理包括哪些步骤？
**答**：主要包括两大部分：清洗（去除特殊字符、多余空白、噪声内容如页眉页脚）和标准化（全角半角统一、标点统一、数字格式统一）。还有一个基础环节是编码处理（确保文件能正确读取）。

### Q2: 为什么预处理对 RAG 很重要？
**答**：预处理直接影响检索质量。如果文档中有噪声内容（如版权声明），检索时可能被误匹配；如果有乱码或特殊字符，切分器可能出错，向量化结果也会异常。预处理可以提高检索准确率 20% 以上。

### Q3: 如何处理编码问题？
**答**：用 try-except 依次尝试常见编码（UTF-8、GBK、GB2312 等）。如果都失败，用 errors='ignore' 或 errors='replace' 强制读取。Python 的 chardet 库可以自动检测编码。

### Q4: 过度清洗会有什么问题？
**答**：过度清洗可能丢失重要信息。比如把所有数字都去掉，就会丢失金额、日期等关键数据；把所有换行都去掉，可能破坏段落结构。正确的做法是只去除明确的噪声，保留有意义的内容。

### Q5: 预处理和切分的顺序是什么？
**答**：先预处理再切分。预处理在前，可以确保切分器拿到的是干净的文本；如果先切分再预处理，可能需要对每个 chunk 分别处理，效率更低，而且跨 chunk 的噪声（如连续空行）更难处理。
