# 文档加载器

## 一句话理解

文档加载器就像一个"万能翻译器"——不管来的是 PDF、Word、TXT 还是 Markdown，它都能翻译成同一种"语言"（Document 对象），让后续的切分和检索模块不需要关心文件格式。

---

## 一、LangChain 的 Document Loader 体系

### 核心思想：统一接口

```
各种格式的文件
  ├── .txt  → TextLoader
  ├── .pdf  → PDFPlumberLoader
  ├── .md   → UnstructuredMarkdownLoader
  ├── .docx → UnstructuredWordDocumentLoader
  └── ...   → 其他 Loader
         │
         ▼
   统一的 Document 对象
   ├── page_content: str    # 文本内容
   └── metadata: dict       # 元数据（来源、页码等）
```

**类比**：Document 对象就像一张标准化的"读书卡片"——不管内容来自哪本书、哪种语言，卡片格式都一样：正面写内容（page_content），背面写来源信息（metadata）。

---

## 二、常用加载器详解

### 1. TextLoader — 纯文本加载器

**适用格式**：.txt、.csv 等纯文本文件

```python
from langchain_community.document_loaders import TextLoader

loader = TextLoader("data.txt", encoding="utf-8")
docs = loader.load()
# docs = [Document(page_content="文件内容...", metadata={"source": "data.txt"})]
```

**特点**：
- 最简单的加载器，直接读取文件内容
- 需要指定编码（中文文件用 `encoding="utf-8"`）
- 整个文件作为一个 Document

### 2. PDFPlumberLoader — PDF 加载器

**适用格式**：.pdf

```python
from langchain_community.document_loaders import PDFPlumberLoader

loader = PDFPlumberLoader("report.pdf")
docs = loader.load()
# 每页生成一个 Document
# docs[0] = Document(page_content="第1页内容", metadata={"source": "report.pdf", "page": 0})
# docs[1] = Document(page_content="第2页内容", metadata={"source": "report.pdf", "page": 1})
```

**特点**：
- 基于 pdfplumber 库，提取效果好
- 自动按页拆分，每页一个 Document
- metadata 中包含页码信息
- 能处理表格、图片中的文字（部分情况）

**PDF 加载器对比**：
| 加载器 | 底层库 | 特点 |
|--------|--------|------|
| PDFPlumberLoader | pdfplumber | 提取效果好，支持表格 |
| PyPDFLoader | pypdf | 轻量，速度快 |
| UnstructuredPDFLoader | unstructured | 功能最全，但较重 |

### 3. UnstructuredMarkdownLoader — Markdown 加载器

**适用格式**：.md

```python
from langchain_community.document_loaders import UnstructuredMarkdownLoader

loader = UnstructuredMarkdownLoader("README.md")
docs = loader.load()
```

**特点**：
- 能识别 Markdown 的结构（标题、列表、代码块等）
- 保留文档的语义结构

### 4. DirectoryLoader — 目录批量加载器

**适用场景**：一次性加载整个目录下的所有文件

```python
from langchain_community.document_loaders import DirectoryLoader, TextLoader

loader = DirectoryLoader(
    path="data/documents/",          # 目录路径
    glob="**/*.txt",                 # 文件匹配模式
    loader_cls=TextLoader,           # 用什么加载器处理每个文件
    loader_kwargs={"encoding": "utf-8"},  # 加载器的额外参数
    show_progress=True,              # 显示进度条
    use_multithreading=True,         # 多线程加速
)

docs = loader.load()
print(f"共加载 {len(docs)} 个文档")
```

**glob 匹配模式**：
| 模式 | 含义 |
|------|------|
| `*.txt` | 当前目录下所有 .txt 文件 |
| `**/*.txt` | 递归匹配所有子目录下的 .txt 文件 |
| `**/*` | 所有文件 |
| `**/*.md` | 所有 .md 文件 |

## 三、Document 对象详解

### 结构

```python
Document(
    page_content="这是文档的文本内容...",   # 必填：文本内容
    metadata={                              # 可选：元数据
        "source": "data/report.pdf",        # 来源文件路径
        "page": 3,                          # 页码（PDF 特有）
        "title": "年度报告",                # 标题
    }
)
```

### metadata 的作用

1. **来源追溯**：回答问题时标注"信息来自哪个文件"
2. **过滤检索**：只在特定文件范围内搜索
3. **去重**：根据 metadata 判断是否是同一份文档

```python
# 按 metadata 过滤检索
results = vectorstore.similarity_search(
    query="公司营收",
    filter={"source": "data/report.pdf"}  # 只在 report.pdf 中搜索
)
```

## 四、多格式加载的统一方案

### 方案一：根据扩展名选择加载器

```python
import os
from langchain_community.document_loaders import (
    TextLoader,
    PDFPlumberLoader,
    UnstructuredMarkdownLoader,
)

def load_file(file_path: str):
    """根据文件扩展名自动选择加载器"""
    ext = os.path.splitext(file_path)[1].lower()

    loader_map = {
        ".txt": lambda: TextLoader(file_path, encoding="utf-8"),
        ".pdf": lambda: PDFPlumberLoader(file_path),
        ".md":  lambda: UnstructuredMarkdownLoader(file_path),
    }

    if ext not in loader_map:
        raise ValueError(f"不支持的文件格式: {ext}")

    return loader_map[ext]().load()
```

### 方案二：用 DirectoryLoader 一次性加载多种格式

```python
from langchain_community.document_loaders import DirectoryLoader

# 加载所有 .txt 文件
txt_loader = DirectoryLoader("data/", glob="**/*.txt", loader_cls=TextLoader)
txt_docs = txt_loader.load()

# 加载所有 .pdf 文件
pdf_loader = DirectoryLoader("data/", glob="**/*.pdf", loader_cls=PDFPlumberLoader)
pdf_docs = pdf_loader.load()

# 合并
all_docs = txt_docs + pdf_docs
```

## 五、面试常问问题

### Q1: LangChain 的 Document Loader 是什么？
**答**：Document Loader 是 LangChain 提供的文档加载接口，负责将各种格式的文件（PDF、TXT、MD 等）转换成统一的 Document 对象。Document 对象包含 page_content（文本内容）和 metadata（元数据，如来源、页码）。不同格式有不同的 Loader 实现，但输出格式统一。

### Q2: Document 对象的 metadata 有什么用？
**答**：三个主要用途：1）来源追溯——回答时标注信息来自哪个文件、第几页；2）过滤检索——指定只在某些文件范围内搜索；3）去重和管理——根据 metadata 判断文档来源。

### Q3: PDF 文件怎么加载？有什么注意事项？
**答**：LangChain 提供了多种 PDF Loader，最常用的是 PDFPlumberLoader（提取效果好）和 PyPDFLoader（轻量快速）。PDF Loader 通常按页拆分，每页生成一个 Document，metadata 中包含页码。注意：扫描版 PDF（图片）需要 OCR 处理，普通 Loader 可能无法提取文字。

### Q4: 如何批量加载一个目录下的所有文档？
**答**：用 DirectoryLoader，指定目录路径和 glob 匹配模式（如 `**/*.txt`）。可以配合 loader_cls 指定用哪个 Loader 处理匹配到的文件。支持多线程加速（use_multithreading=True）。

### Q5: 不同格式的文档加载后怎么统一处理？
**答**：LangChain 的所有 Loader 都输出 Document 对象格式，所以加载后就不需要关心原始格式了。后续的清洗、切分、向量化都操作 Document 对象。关键是在 metadata 中记录来源信息，方便后续追溯。
