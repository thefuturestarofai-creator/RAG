# Linux 终端操作基础

> 学习目标：掌握 Linux/Mac 终端的常用命令，能在命令行中完成文件导航、文件操作、内容查看等基本任务。
> 这些技能是使用服务器部署 RAG 系统的必备基础。

---

## 0. 什么是终端？

**通俗理解**：终端就像是电脑的"后门"，可以直接和操作系统对话。

- 图形界面（GUI）：你用鼠标点点点，系统帮你执行命令
- 终端（Terminal/CLI）：你直接输入文字命令，系统直接执行

**为什么要学终端？**
- 服务器通常只有终端，没有图形界面
- 很多开发工具只能在终端中使用
- 批量操作文件时，终端比鼠标快 100 倍
- 自动化脚本（如部署 RAG 系统）必须用终端

**打开终端的方式**：
- **Windows**：Git Bash、PowerShell、WSL（推荐 Git Bash）
- **Mac**：Terminal（在"应用程序→实用工具"中）
- **Linux**：Ctrl+Alt+T

## 1. 文件导航命令（最常用）

### 1.1 `pwd` —— 我在哪里？

```bash
# pwd = Print Working Directory（打印当前工作目录）
pwd
# 输出示例：/home/user/projects/rag-system
```

**类比**：就像你打开手机地图，显示"你当前在北京市海淀区"。

### 1.2 `ls` —— 这里有什么？

```bash
# ls = List（列出当前目录的文件和文件夹）
ls

# 常用选项：
ls -l    # 长格式显示（权限、大小、日期等详细信息）
ls -a    # 显示隐藏文件（以 . 开头的文件）
ls -lh   # 长格式 + 文件大小用人类可读的方式显示（KB/MB）
ls -la   # 组合使用：显示所有文件的详细信息
```

**输出示例**：
```
drwxr-xr-x  5 user staff  160 Jun  1 10:00 .
drwxr-xr-x  3 user staff   96 May 30 09:00 ..
-rw-r--r--  1 user staff  245 Jun  1 10:00 main.py
-rw-r--r--  1 user staff 1.2K Jun  1 10:00 README.md
drwxr-xr-x  2 user staff   64 Jun  1 10:00 data
```

**解读 `-l` 输出**：
- 第1列：文件类型和权限（`d`=目录，`-`=文件）
- 最后3列：修改时间、文件名
- 中间列：文件大小

### 1.3 `cd` —— 我要去哪里

```bash
# cd = Change Directory（切换目录）

cd /home/user/projects    # 绝对路径：从根目录开始的完整路径
cd projects               # 相对路径：相对于当前目录
cd ..                     # 回到上一级目录
cd ../..                  # 回到上两级目录
cd ~                      # 回到用户主目录（~ = home）
cd -                      # 回到上一次所在的目录
cd                        # 不带参数，也回到主目录（同 cd ~）
```

**类比**：
- 绝对路径 = 完整地址："北京市海淀区中关村大街1号"
- 相对路径 = 相对位置："往前走两个路口"

## 2. 文件操作命令

### 2.1 `mkdir` —— 创建文件夹

```bash
# mkdir = Make Directory（创建目录）

mkdir my_project                    # 创建单个目录
mkdir -p projects/rag/src           # -p 选项：自动创建多级目录（如果父目录不存在）
mkdir -p data/{raw,processed,output}  # 一次创建多个子目录
```

### 2.2 `rm` —— 删除文件/目录

```bash
# rm = Remove（删除）

rm file.txt               # 删除文件（会提示确认）
rm -f file.txt            # -f = Force，强制删除，不提示
rm -r my_folder           # -r = Recursive，删除目录及其所有内容
rm -rf my_folder          # 组合：强制递归删除（最危险的命令！）

# 安全做法：先用 ls 确认要删除的内容
ls my_folder/             # 先看看里面有什么
rm -r my_folder           # 再删除
```

**警告**：`rm -rf /` 会删除系统所有文件！绝对不要执行！

### 2.3 `cp` —— 复制文件

```bash
# cp = Copy（复制）

cp file.txt file_backup.txt              # 复制文件
cp -r my_folder my_folder_backup         # -r：复制目录（必须加 -r）
cp file.txt /home/user/backup/           # 复制到指定目录
cp *.txt /home/user/backup/              # * 是通配符，匹配所有 .txt 文件
```

### 2.4 `mv` —— 移动/重命名文件

```bash
# mv = Move（移动）

mv file.txt /home/user/backup/           # 移动文件到另一个目录
mv old_name.txt new_name.txt             # 重命名（本质是移动到同目录的新名字）
mv *.log logs/                           # 移动所有 .log 文件到 logs 目录
```

## 3. 查看文件内容

### 3.1 `cat` —— 查看整个文件

```bash
# cat = Concatenate（连接，但常用来查看文件内容）

cat file.txt                    # 显示文件全部内容
cat file1.txt file2.txt         # 依次显示多个文件的内容
cat -n file.txt                 # -n：显示行号
```

**适用场景**：查看小文件（几十行以内）

### 3.2 `tail` —— 查看文件末尾

```bash
# tail：查看文件的最后几行

tail file.txt                   # 默认显示最后 10 行
tail -n 20 file.txt             # 显示最后 20 行
tail -f app.log                 # -f = Follow，实时跟踪文件新增内容（看日志神器！）
tail -f logs/rag.log            # 实时查看 RAG 系统的日志
```

**适用场景**：查看日志文件（日志通常很大，你只关心最新的）

### 3.3 `head` —— 查看文件开头

```bash
# head：查看文件的前几行

head file.txt                   # 默认显示前 10 行
head -n 5 file.txt              # 显示前 5 行
```

### 3.4 `grep` —— 搜索文件内容

```bash
# grep = Global Regular Expression Print（全局正则表达式打印）

grep "error" app.log                    # 在 app.log 中搜索包含 "error" 的行
grep -i "error" app.log                 # -i = Ignore case，忽略大小写
grep -n "error" app.log                 # -n：显示行号
grep -r "API_KEY" .                     # -r = Recursive，递归搜索当前目录所有文件
grep -rn "error" --include="*.py" .     # 只搜索 .py 文件
grep -c "error" app.log                 # -c = Count，只显示匹配的行数
grep -A 3 "error" app.log              # -A = After，显示匹配行及其后 3 行
grep -B 3 "error" app.log              # -B = Before，显示匹配行及其前 3 行
```

**类比**：grep 就像 Ctrl+F，但功能更强大，可以跨文件搜索。

**实际使用示例**：
```bash
# 在 RAG 项目中搜索所有 Python 文件中的 "embedding"
grep -rn "embedding" --include="*.py" .

# 搜索日志中今天的错误
grep "2024-06-01.*ERROR" logs/app.log

# 搜索 API Key 是否泄露在代码中（安全检查）
grep -rn "sk-" --include="*.py" .
```

## 4. 环境变量

### 4.1 什么是环境变量？

**通俗理解**：环境变量就像是系统的"便签纸"，你可以在上面写一些信息，程序运行时可以读取这些信息。

**常见用途**：
- 存储 API Key（不让它出现在代码中）
- 配置程序运行参数
- 指定文件路径

### 4.2 查看环境变量

```bash
# 查看所有环境变量
env

# 查看特定环境变量
echo $HOME                  # 显示用户主目录路径
echo $PATH                  # 显示系统搜索命令的路径列表
echo $OPENAI_API_KEY        # 显示 OpenAI API Key（如果设置了的话）
```

### 4.3 设置环境变量

```bash
# 临时设置（当前终端会话有效，关闭终端就失效）
export OPENAI_API_KEY="sk-your-key-here"
export OPENAI_BASE_URL="https://api.openai.com/v1"

# 验证设置
echo $OPENAI_API_KEY

# 在 Python 中读取环境变量
python -c "import os; print(os.environ.get('OPENAI_API_KEY'))"
```

### 4.4 永久设置环境变量

**Linux/Mac**：在 `~/.bashrc` 或 `~/.zshrc` 文件末尾添加：
```bash
export OPENAI_API_KEY="sk-your-key-here"
export OPENAI_BASE_URL="https://api.openai.com/v1"
```

然后执行：
```bash
source ~/.bashrc    # 让配置立即生效
```

**Windows（Git Bash）**：在 `~/.bashrc` 中添加同样的内容

### 4.5 为什么用环境变量存储 API Key？

```python
# 错误做法：API Key 写在代码里（会被上传到 GitHub，造成泄露！）
api_key = "sk-1234567890abcdef"

# 正确做法：从环境变量读取
import os
api_key = os.environ.get("OPENAI_API_KEY")
```

## 5. pip/python 命令行运行

在终端中运行 Python 脚本和管理包是日常开发的基本操作。

### 5.1 运行 Python 脚本

```bash
# 运行 Python 脚本
python main.py

# 运行并传入参数
python main.py --query "什么是RAG？" --top-k 5

# 运行 Python 单行代码（快速测试）
python -c "print('Hello, RAG!')"

# 运行 Python 单行代码（导入模块测试）
python -c "import json; print(json.dumps({'key': 'value'}))"

# 进入 Python 交互式环境（REPL）
python
# >>> 输入 Python 代码，按回车执行
# >>> 输入 exit() 或按 Ctrl+D 退出
```

### 5.2 pip 包管理命令

```bash
# 安装包
pip install langchain

# 安装指定版本
pip install openai==1.12.0

# 从 requirements.txt 安装所有依赖
pip install -r requirements.txt

# 查看已安装的包
pip list

# 查看某个包的信息
pip show langchain

# 导出当前环境的所有依赖
pip freeze > requirements.txt

# 升级包
pip install --upgrade langchain

# 卸载包
pip uninstall langchain

# 使用国内镜像加速下载
pip install langchain -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 5.3 常见的开发工作流

```bash
# 典型的 RAG 项目开发流程：

# 1. 进入项目目录
cd ~/projects/my-rag-project

# 2. 激活虚拟环境
source venv/bin/activate          # Linux/Mac
# venv\Scripts\activate           # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 运行项目
python main.py

# 5. 运行测试
python -m pytest tests/

# 6. 查看运行日志（另一个终端窗口）
tail -f logs/app.log
```

```bash

```

## 7. 面试常问问题

### Q1：`grep` 和 `find` 命令有什么区别？

**参考答案**：
- `find` 用于**查找文件**：`find . -name "*.py"` 查找所有 Python 文件
- `grep` 用于**搜索文件内容**：`grep "error" *.py` 在 Python 文件中搜索 "error"
- 组合使用：`find . -name "*.py" -exec grep "error" {} \;` 先找文件，再搜索内容

### Q2：什么是环境变量？为什么 API Key 要用环境变量存储？

**参考答案**：
- 环境变量是操作系统级别的键值对存储，程序可以读取
- API Key 用环境变量存储的原因：
  1. **安全性**：不会被写入代码仓库（避免泄露）
  2. **灵活性**：不同环境（开发/测试/生产）可以用不同的 Key
  3. **规范性**：是行业标准做法，12-Factor App 原则之一

### Q3：`tail -f` 命令在实际工作中有什么用途？

**参考答案**：
- 实时监控日志文件：`tail -f logs/app.log`
- 调试程序：运行程序的同时，另一个终端用 `tail -f` 看日志
- 监控 RAG 系统运行状态：观察请求处理、错误信息等
- 配合 `grep` 过滤：`tail -f logs/app.log | grep "ERROR"` 只看错误日志

## 9. 练习建议

1. **每天练习**：用终端代替图形界面操作文件，坚持一周就熟练了
2. **善用 Tab 补全**：输入命令或路径的前几个字母，按 Tab 自动补全
3. **善用历史命令**：按上下箭头可以翻看之前执行过的命令
4. **不要怕犯错**：终端命令大多可以撤销（除了 `rm`，要小心）
5. **记住 `man` 命令**：`man ls` 可以查看 `ls` 命令的完整文档
