# API 概念

> API（Application Programming Interface，应用程序编程接口）是不同软件之间"对话"的桥梁。RAG 系统中，你调用 OpenAI 的 API 来获取 Embedding 和 LLM 回答，调用向量数据库的 API 来存储和检索文档。理解 API，是开发 RAG 应用的前提。

---

## 1. 用"餐厅服务员"来理解 API

想象你去餐厅吃饭：

- **你**（客户端）：想吃宫保鸡丁
- **厨房**（服务端）：能做宫保鸡丁，但你不能直接进厨房
- **服务员**（API）：你告诉服务员"来一份宫保鸡丁"，服务员传达给厨房，做好后端给你

**API 就是"服务员" —— 你不需要知道厨房怎么运作，只需要知道怎么跟服务员沟通（API 文档）。**

---

## 2. 什么是 RESTful API？

RESTful API 是目前最流行的 API 设计风格。核心思想：**一切皆资源，用 HTTP 方法操作资源**。

### 餐厅菜单类比

| 操作 | HTTP 方法 | URL | 餐厅类比 |
|------|----------|-----|---------|
| 查看所有菜品 | GET | /menu | 看菜单 |
| 查看某道菜 | GET | /menu/1 | 看宫保鸡丁的详情 |
| 添加新菜品 | POST | /menu | 厨师研发新菜 |
| 修改菜品 | PUT | /menu/1 | 改宫保鸡丁的做法 |
| 删除菜品 | DELETE | /menu/1 | 从菜单上撤掉宫保鸡丁 |

### RAG 系统中的 RESTful API

| 操作 | HTTP 方法 | URL 示例 |
|------|----------|---------|
| 搜索文档 | POST | /api/search |
| 获取文档 | GET | /api/documents/123 |
| 上传文档 | POST | /api/documents |
| 删除文档 | DELETE | /api/documents/123 |
| 调用 LLM | POST | /api/chat |

## 3. localhost 和端口号

### localhost 是什么？

`localhost` = `127.0.0.1` = **你自己的电脑**。开发时，服务器运行在本地，用 localhost 访问。

### 端口号是什么？

一台电脑可以运行多个服务，端口号用来区分它们。就像一栋楼有多个房间，门牌号就是端口号。

```
你的电脑 (localhost)
├── :3000  →  前端页面（React）
├── :8000  →  后端 API（FastAPI）
├── :8080  →  另一个服务
├── :5432  →  数据库（PostgreSQL）
└── :6333  →  向量数据库（Qdrant）
```

### 常见端口号

| 端口 | 常见用途 |
|------|---------|
| 80 | HTTP 默认端口 |
| 443 | HTTPS 默认端口 |
| 3000 | 前端开发服务器 |
| 5000 | Flask 默认端口 |
| 8000 | FastAPI 默认端口 |
| 8080 | 备用 HTTP 端口 |

### URL 中的端口

```
http://localhost:8000/api/search
│       │         │     │
│       │         │     └── API 路径
│       │         └── 端口号
│       └── 主机地址（本地电脑）
└── 协议
```

## 4. 用浏览器开发者工具查看请求

这是**非常实用的技能**，可以帮你理解 API 的工作方式。

### 打开开发者工具

- Chrome/Edge: `F12` 或 `Ctrl+Shift+I`
- Firefox: `F12`

### 查看网络请求

1. 打开开发者工具，切换到 **Network**（网络）标签
2. 访问一个网站（比如 https://api.github.com/users/octocat）
3. 在 Network 面板中点击对应的请求
4. 查看：
   - **Headers**: 请求方法、URL、请求头
   - **Payload**: 请求体（POST 请求的数据）
   - **Response**: 服务器返回的数据
   - **Status**: 状态码

### 实际操作

```
1. 打开浏览器，按 F12
2. 点击 Network 标签
3. 地址栏输入: https://api.github.com/users/octocat
4. 在 Network 面板中找到这个请求
5. 点击它，查看：
   - Request URL: https://api.github.com/users/octocat
   - Status Code: 200 OK
   - Response: {"login": "octocat", "name": "The Octocat", ...}
```

## 5. 用 Python 调用 API（预告）

```python
import requests        # Python 最流行的 HTTP 库

# GET 请求
response = requests.get("https://api.github.com/users/octocat")
print(response.status_code)    # 200
print(response.json())         # 解析 JSON 响应

# POST 请求（模拟调用 OpenAI API）
response = requests.post(
    "https://api.openai.com/v1/chat/completions",
    headers={"Authorization": "Bearer sk-xxx"},
    json={"model": "gpt-3.5-turbo", "messages": [...]}
)
```

## 7. 在 RAG 项目中的应用场景

| API 概念 | RAG 中的具体应用 |
|---------|-----------------|
| RESTful API | LangChain 的 `.invoke()` 方法底层就是调用 REST API |
| POST 请求 | 向 OpenAI API 发送提问和上下文 |
| JSON 格式 | API 请求体和响应体的标准格式 |
| localhost | 本地运行的 FastAPI/Flask 后端服务 |
| 端口号 | 本地向量数据库（如 Chroma:8000） |
| 认证 | API Key 放在 Authorization 请求头中 |
| 状态码 | 判断 API 调用是否成功、是否需要重试 |

### RAG 系统的典型 API 调用链

```
用户提问 "什么是RAG？"
    │
    ▼
[1] 调用 Embedding API (POST)
    → 把问题转成向量 [0.1, 0.2, ...]
    │
    ▼
[2] 调用向量数据库 API (POST /search)
    → 用向量检索相关文档
    │
    ▼
[3] 构造 Prompt = 问题 + 检索结果
    │
    ▼
[4] 调用 LLM API (POST /chat/completions)
    → 生成最终回答
    │
    ▼
返回回答给用户
```

## 8. 面试官常问问题

**Q1: 什么是 API？用你自己的话解释。**

A: API 就是软件之间的"服务窗口"。你不需要知道内部怎么实现，只需要按照规定的格式发送请求，就能拿到结果。就像去银行办业务：你填表（请求），柜员处理（服务端），给你结果（响应）。

**Q2: RESTful API 是什么？有什么特点？**

A: REST 是一种 API 设计风格，核心特点：一切皆资源（用 URL 表示）、用 HTTP 方法操作（GET/POST/PUT/DELETE）、无状态（每次请求独立）、数据格式用 JSON。

**Q3: GET 和 POST 在实际使用中有什么区别？**

A: GET 参数在 URL 中（可见、有长度限制、可缓存），适合查询。POST 参数在请求体中（不可见、无长度限制），适合提交数据。调用 LLM 用 POST，因为消息内容可能很长且包含敏感信息。

**Q4: 什么是 API Key？为什么需要它？**

A: API Key 是访问 API 的"钥匙"，用于身份验证和计费。没有 Key 就无法调用（会返回 401）。Key 要保密，不能写在代码里或提交到 Git。

**Q5: 如何调试 API 调用问题？**

A: 1) 用浏览器开发者工具查看请求/响应；2) 用 Postman 工具手动发请求测试；3) 用 Python 的 requests 库打印 status_code 和 response.text；4) 检查状态码定位问题（401=Key错，429=限流，500=服务器错误）。
